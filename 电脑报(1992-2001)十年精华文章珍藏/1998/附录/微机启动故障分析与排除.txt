微机启动故障分析与排除

唐奂知
1998年 第52期 36版

    微机给人们的工作、生活和学习带来了极大的方便和乐趣，同时也给电脑用户带来了不少的烦恼，其中，微机的故障便是众多电脑用户的主要烦恼之一。在微机的各种故障中，微机不能启动是电脑用户普遍面临的问题，如果处理不当，轻则丢失微机系统的重要数据，重则会造成微机硬件的损坏。因此，如何检修微机不能启动的故障，是大多数电脑用户普遍关心的问题。下面结合大量的维修实践，详尽地介绍微机的各种不能启动的原因及其相应的检修方法。
#1    一、微机启动过程分析
    为了更清楚地了解微机的各种启动故障，正确区分造成微机无法启动的各种不同的原因，从而能够快速准确地将故障修复，先对微机正常情况下的启过程作一个了解。一台正常的微机的启动过程大致可以分为如下几个阶段：
    第一阶段：电源工作，CPU复位。开启微机的电源开关，微机内部的开关电源首先通电工作，并输出信号将微机的CPU复位（RESET）。
    第二阶段：硬件检查与设置检查。当CPU复位成功后，微机将执行内部固化的程序，对微机的硬件（包括内存）进行检查，并将检查结果与微机的CMOS设置进行对比。
    第三阶段：读取分区记录和引导记录。当微机检查到硬件正常并与CMOS设置相符后，将首先检查软盘驱动器中是否插有软盘，如果有，将优先通过软盘启动；如果软驱中没有软盘，微机将读取硬盘的分区记录和主引导记录。
    第四阶段：读取DOS引导记录。微机正确读取分区记录和主引导记录后，将进一步读取DOS引导记录，将磁盘操作系统（DOS）将入内存。
    第五阶段：装载系统隐含文件。读取DOS引导记录后，微机将DOS系统的隐含文件（IO.SYS或IBMBIO.SYS、MSDOS.SYS或IBMDOS.COM）装入内存。
    第六阶段：执行DOS的引导过程。系统隐含文件装载完成，微机将执行系统隐含文件，并执行系统配置文件（CONFIG.SYS）。
    第七阶段：装入命令管理程序（COMMAND.COM）。系统装载命令管理程序，以便对系统的各各操作命令进行协调管理。
    第八阶段：执行批处理文件。如果启动软盘或硬盘根目录下有自动批处理文件（AUTOEXEC.BAT），微机将一步一步地执行批处理文件中的各条命令。
    第九阶段：启动完成，执行具体的工作。
#1    二、检修中应当注意的问题
    微机不能启动的故障原因是多方面的，有由于硬件损坏造成的不能启动的硬故障，也有由于CMOS设置错误、硬盘主引导扇区数据遭受破坏、DOS引导记录遭受破坏等软故障。相对而言，微机不能启动多数是由于软故障所引起的。检修微机不能启动的硬故障，需要操作者应该具备一定的电子技术的硬件知识，还应当拥有一定的电工工具，如一字螺丝刀、十字螺丝刀、电烙铁、万用表等基本工具，对于主板、CPU等故障，还必须拥有逻辑笔、示波器等高级检修工具，否则，不但不能修复故障，反而会将故障范围扩大，因此建议普通用户最好不要开机检修微机硬件。当然，对怀疑有故障的微机组件进行替换检查，对可能接触不良的插头、插座进行必要的处理是完全可以的。
    检修微机不能启动的软故障，是大多数电脑用户都可以进行的，但你手中必须拥有适当的用于电脑维修的工具软件，检修过程中，根据不同的故障选用相应的工具软件进行处理，一个好的工具软件将使维修工作轻松快捷、安全可靠、事半功倍。检修时应该先软后硬，先易后难，先外后内。在更改原有数据之前，应当对原来的内容作好记录，或者通过相应软件作好数据备份，如果改写数据后仍然不能使故障修复，应当及时将数据还原。切忌在不对原有数据进行备份的情况下，到处修改数据，以免增加新的故障，使检修陷入困境。在后面的各个部分中，将详细介绍微机启动的各个阶段可能出现的各种故障及其检修方法。由于大多数微机都是通过硬盘启动微机，因此，后面的分析与检修方法都以硬盘启动为基础。
#1    三、启动的各个阶段可能出现的故障分析
    第一阶段可能出现的故障与检修：
    在启动的第一阶段，操作者开启微机的电源开关，微机的开关电源得电工作，CPU复位。在这一阶段，必须保证微机外部的电源（稳压电源和UPS等）有输出，在微机的电源插头与外部电源插接可靠，微机的电源线没有断路点，微机内部的开关电源及开关电源与微机主板的连线、插头、插座等正常，还应当保证微机主板和CPU正常。如果微机外部的电压较低，插头、插座接触不良，微机内部的电源开关或开关电源损坏，连线断路，插头、插座之间动，微机主板或CPU损坏等任意一种原因都将导致微机启动失败。如果在第一阶段出现故障，微机上的各种指示灯常常无法点亮，扬声器无任何声响。
    此时应当首先检查微机的外部原因，听一听微机开关电源的小轴流风扇是否工作，如果小轴流风扇工作正常，基本可以肯定微机外部电源及内部的开关电源正常（当然，少数情况下也有开关电源损坏的实例），然后关掉微机电源，仔细检查微机内部的各种电源插头、插座是否有松动，连线是否有断路；如果微机开关电源内的小轴流风扇没有工作，应重点检查微机外部原因，检测给微机供电的交流稳压电源是否正常，UPS的输出插座是否有220V的交流电，再将微机的电源插头从外部稳压电源或UPS插座上取下，用万用表检测微机电源线插头两边的插线片的电阻值是否正常（测量时应开启微机的电源开关，用万用表的1千欧姆档测试，阻值应从几千欧慢慢变到几十千欧为正常），接下来检查开关电源。在保证微机外部及内部开关电源正常的情况下，最后才检查主板及CPU等。普通的用户最好不对开关电源的内部进行检修，但有条件的用户可以采用替换的方法确定故障部位，即用好的同类型的开关电源替换怀疑有故障的开关电源，用好的同类型的带CPU的主板替换怀疑有故障的主板（含CPU）。如果确定是开关电源损坏，可以请技术较好的修理彩电的师傅检修，或者更换新的开关电源。如果确定是主板或CPU损坏，最好换新，换新的同时，你可以根据自己的档次，确定是否升级。
    由于这一阶段的故障常涉及到220V的交流电，检修时尤其应当注意安全，以防触电。对微机内部的检查应当十分小心，否则将会导致故障范围扩大，造成更大的损失。没有经验的用户最好请专业维修人员进行检修。
#1    四、第二阶段可能出现的故障
    微机启动的第二阶段，将对系统的主要硬件进行检查，并将检查结果与系统的CMOS设置进行比较。这一阶段应当保证各种硬件完好无损，硬件与硬件相应的驱动卡的连线正确而且可靠，主板上的各种插卡与扩展糟接触良好，各种硬件的参数应当与CMOS设置的参数一致，否则，将可能导致启动失败。在这一阶段启动受阻，如果显示卡、显示器等正常，微机显示器将会有相应的提示信息。此时应该充分利用微机给出的提示信息进行检修，如果对提示的英文信息不能理解，或无法弄懂故障代码的含义，一定要查阅资料弄清楚，切不可盲目处理。
    这一阶段可能出现的故障现象有：
    1.磁盘引导失败。
    2.微机忽略硬盘的存在。
    3.提示故障代码1701、1780、1781、1790、1791等。
    4.给出提示信息“RUN CMOS SETUP Press<F1>TO RESUME”、“C：DRIVE ERROR PERSS<F1>TO RESUME”、“NO BASIC”、“Hard Disk 0 failure”、“HDD contorller failure press<F1>resume”、“FDD contorller failure press<F1>resume”等。
    造成这些故障的原因主要是CMOS设置错误或丢失、硬盘与硬盘驱动卡排线联接错误或松动、软驱与软驱适配卡联接异常、硬盘或软盘电源线导演或供电电压偏离正常值、驱动器硬件损坏等。对常见故障提示信息的处理方法如下：
    1.提示“RUN CMOS SETUP PRESS<F1>TO RESUME”的原因及处理方法
    这种提示主要是由于微机的CMOS设置错误，通常在提示信息前还有一行提示信息说明了需要运行CMOS设置的原因：
    （1） “CMOS display type mismatch”
    CMOS设置的显示器类型与实际检测到的显示器类型不匹配，重新运行标准CMOS设置，修改显示器类型
    （2） “CMOS memory size mismatch”
    CMOS设置的存储器容量与实际检测到的存储器容量不匹配，重新运行高级CMOS设置，修改存储器容量。
    （3） “Keyboard error”
    CMOS设置的键盘类型与实际检测到的键盘类型不符，重新设置键盘类型。
    2.提示“CMOS Battary state LOW”的原因及处理方法
    这种揭示主要是由于系统中用于保存CMOS设置的电池电压不足或电池损坏，检修时先关掉微机电源，用电压表测量CMOS电池电压，如果在3.6V左右，则应检查电与主板之间的线路。如果电池电压远远低于3.6V，则应更换电池，如果更换电池后不久又出现这种故障提示，还应检查电池的充电电路。
    3.提示“FDD contorller failure press<F1>resume”的原因及处理方法
    这种提示主要是微机对软盘适配器检测失败，此时应该重点检查软盘驱动器的电源线、信号线的联接是否正常，或者是根本就没有安装软驱，却在CMOS设置中设置了软驱。最后检查软驱适配卡。
    4.提示“HDD contorller failure press<F1>resume”的原因及处理方法
    这种提示主要是微机对硬盘检测失败，此时应该重点检查硬盘驱动器的电源线、信号线是否联接正常，或者是根本就没有安装硬盘，却在CMOS中设置了硬盘。最后检查硬盘驱动器适配卡。
    5.同时提示了上面3、4两种信息“FDD contorller failure HDD contorller failure press<F1>resume”的原因及处理方法
    同时提示软盘和硬盘控制失败，应当首先检查控制软盘和硬盘的多功能卡是否有故障、多功能卡与主板是否有接触不良，将多功能卡插入另一个扩展槽试一试，或用一块完好的多功能卡替换再试一试。
    6.提示“C:drive error press<F1>to resume”的原因及处理方法
    这种提示主要是微机对硬盘驱动器检测时，未收到任何响应信号，或硬盘参数设置错误。此时应检查硬盘参数设置，再检查硬盘的电源线和信号线是否联接正常，线路中是否有断路。
#1    五、第三阶段可能出现的各种故障
    这一阶段主要是读取硬盘的分区信息和主引导记录，在保证硬盘的参数与CMOS设置相同、硬盘及其驱动卡正常的前提下，还应当保证硬盘分区信息和主引导记录未受破坏。
    如果硬盘分区信息或主引导记录遭受破坏，将有可能显示如下几种提示信息：
    1.DRIVE NOT READY ERROR Insert boot diskette in A:。
    2.NO ROM BASIC SYSTEM HALTED。
    3.Invalid parition table。
    4.Invalid drive specification。
    5.missing operation system。
    6.微机硬件榆查通过后，无任何提示信息并死机。
    对上述几种故障的检修方法如下：
    1.提示“DRIVE NOT READY ERROR Insert boot diskette in A:”的原因及处理方法这种故障主要是由于硬盘的分区记录遭到破坏，或硬盘在格式化时未向硬盘传输系统文件，或在未安装硬盘的微机的软驱中没有插入软盘等原因导致的。检修时先将一张可以启动微机的系统盘插入软驱，如果可以启动，但转入硬盘时揭示“Invaild drive specification”（无效的驱动器定义），不能操作硬盘。
    这种故障主要是硬盘分区信息错误或分区结束标志（1FE、1FF的55AA）丢失，此时可以用Norton Utilites 8.0中的DISKEDIT软件查看并修改硬盘分区记录的分区结束标志。如果用软盘也不能启动微机，应当检查控制软驱和硬盘的多功能卡。
    2.提示“NO ROM BASIC SYSTEM HALTED”的原因及处理方法
    这种故障主要是由于硬盘主引导扇区的活动分区标志（1BE的80H）丢失或分区信息严重破坏，首先用Norton Utilites 8.0中的DISKEDIT软件查看并修改硬盘分区记录的活动分区标志（1BE的80H）。如果发现硬盘的分区记录损坏严重或更改活动分区标志示（1BE的80H）后仍然有相同的故障提示，可能用杀毒软件KV300对硬盘主引导扇区进行修复。
    3.提示“Invalid parition table”的原因及处理方法
    这种故障是因为硬盘的分区记录遭到破坏，检修时应当首先查看硬盘的分区记录，看分区记录的活动分区标志（1BE的80H）和分区结束标志（1FE、FF的55AA）是否丢失，如果丢失，用DISKEDIT将其改正。
    4.提示“Invalid drive specification”的原因及处理方法
    这是由于CMOS设置的硬盘参数与实际的硬盘参数符，或硬盘低级格式化后未分区，或分区后未经高级格式化，或硬盘的分区记录遭到破坏。对于正常使用的微机突然出现这种不能启动故障，应当首先考虑硬盘分区记录遭受破坏，处理方法同“Invalid parition table”故障。
    5.提示“missing operation system”的原因及处理方法
    在这一阶段出现这种提示，是由于硬盘主引导记录严重破坏或主引导结束标志（080、081的55AA）丢失，应当注意，在以后几个启动阶段也可能出现这种提示，因此，出现“missing operation system”提示并不是微机读取硬盘分区记录和主引导记录阶段的特有故障。微机出现这种故障主要是由于下列三种原因之一引发的：
    （1）硬盘根目录下的两个系统文件（IO.SYS【IBMBIO.COM】或MSDOS.SYS【IBMDOS.COM】）之一被破坏，或被误删除。
    （2）硬盘DOS引导记录（BOOT）严重破坏，或DOS引导记录的结束标志（01FE、01FF的55AA）丢失。
    （3）硬盘主引导数据严重破坏，或主引导结束标示（080、081的55AA）丢失。
    遇到这种故障，应该按照上述（1）—（3）的顺序检查和处理，以避免更大损失。先用软盘启动微机，再转入硬盘，用DIR/AS命令查看硬盘（C）的根目录是否有DOS的两个系统文件，如果没有，用软盘的SYS C：命令向硬盘传输系统文件；如果有两个系统文件，可以用NU8的NDD试着修复硬盘DOS引导记录（BOOT），最后用DISKEDIT修改硬盘主引导记录的结束标志（080、081的55AA）。
    6.微机自检后无任何揭示，电脑死机的原因及处理方法
    发生这种故障的原因是硬盘主引导程序被严重破坏或主引导程序的开始三个字节（FA 33 CO）丢失，用软盘启动后可以进入硬盘。检修的最简办法是用软盘启动微机后用DOS的FDISK命令加MBR参数修复，即在相应的路径下键入FDISK/MBR即可。
#1    六、第四阶段可能出现的各种故障
    当微机读取硬盘主引导扇区正常后，将读取硬盘的DOS引导记录，如果硬盘DOS引导记录错误，微机将可能显示如下提示信息：
    1.missing operation system。
    2.Error loading operation system。
    3.Non system disk or disk error。
    4.Disk boot failure。
    对上述故障的检修方法如下：
    1.微机自检后提示“missing operation system”并死机的原因及处理方法详见启动第三阶段所述。
    2.微机提示“Error loading operation system”的原因及处理方法
    本故障主要是由于硬盘DOS引导记录损坏，或两个系统隐含文件发生错误，检修时可以用NDD修复系统DOS引导记录（BOOT），如果故障未能排除，则用DOS命令SYS C：向硬盘传输两个系统隐含文件。
    3.微机提示“Non system disk or disk error”的原因及处理方法
    这种故障通常是由于硬盘的DOS系统文件IO.SYS（IBMBIO.COM）或MSDOS.SYS（IBMDOS.COM）遭到破坏，或是刚进行过高级格式化，且格式化时忘了传输系统文件。检修时可以先用软盘启动微机后，再用DOS的SYS C：命令向硬盘传输系统文件即可。
    4.微机提示“Disk boot failure”的原因及处理方法
    本故障的产生原因及检修方法与“Error loading operation system”类似，请参考前述第2条。
#1    七、第五阶段可能出现的各种故障
    这一阶段微机装载磁盘操作系统的两个隐含文件IO.SYS（或IBMBIO.COM）和MSDOS.SYS（或IBMDOS.COM），如果两个隐含文件不在硬盘DOS活动分区的根目录下，或不在根目录的最前端，或两个陷含文件中的任意一个受到破坏，或C：盘分区表损坏使DOS引导程序无法找到两个系统隐含文件，都将导致微机启动失败。
    在这一阶段可能出现的帮障提示有：
    1.Non-system disk or disk error
    Replace and starike any key when ready。
    2.Error loading operation system。
    3.Disk boot failure。
    4.Nissing operating system。
    5.微机自检后死机
    出现以上的故障提示，除了与DOS操作系统的两个系统隐含文件密切相关外，有时还与硬盘的DOS引导记录有关。当微机出现上述故障时，应先考虑并检查和处理系统隐含文件故障，如果不能解决，再检查DOS引导记录。
    处理因系统隐含文件丢失，或系统隐含文件遭受破坏，或系统隐启文件在硬盘根目录中的位置错乱等引起的各种故障的最简单有效的方法是用DOS的SYS系统传送命令。其具体方法是先用软盘启动微机，再键入SYS C：命令，如果在分区时采用了其它逻辑盘作为DOS启动盘，则驱动器盘符应该是相应的逻辑盘。如果硬盘是采用了扩容措施的压缩盘，系统应当传送到宿主盘（如宿主盘符为H，命令格式则为SYS H：）。系统文件传送成功后，微机将提示：“system tyansferred”，如果出现其它提示，应该根据提示寻找原因。
    对于上述各种提示的具体原因及处理方法，在前面微机启动第四阶段的故障分析中已经介绍。
    不会排列为空缺，因此用PCTOOLS的文件服务功能查看根目录的文件显示MSDOS.SYS为第二个文件，使用户的检修工作陷入误区。MSDOS.SYS文件不在根目录的第二个目录项，也许就是硬盘无法启动的根本原因。在PCTOOLS的磁盘服务功能（F3）下，可以直接启动编辑功能将硬盘根目录的第二个目录项的第一个字节由原来的“E5”改为“4D”（MSDOS.SYS第一个字母“M”的ASCII码），将第三个目录项的第一个字节由“4D”改为“E5”就可以修复故障。
    由于硬盘不能启动，长期只能用软盘启动，并且许多程序在运行过程中常常要读取软盘的命令解释程序COMMADN.COM，加快了软驱及启动软盘的磨损，导致后来软盘也不能启动。实际检修中，遇到本阶段出现的各种故障提示，都可以先用DOS的SYS C：命令进行简单的处理，使用该命令是有有益而无害的。DOS4.0以后的系统传送命令SYS比DOS3.3或以前版本有较大改善，对修复微机因DOS的两个系统隐含微机和命令管理程序COMMAND.COM所导致的各种故障都有立杆见影的效果。
    微机不能启动的分析与检修
#1    八、第六阶段可能出现的各种故障
    系统隐含文件装载完成后，微机将执行系统隐含文件，开始DOS的引导，并执行系统配置文件（CONFIG.SYS）。当微机装入系统的第一个隐含文件时，引导过程就已经开始，在引导过程中，微机还将执行系统配置文件CONFIG.SYS。这一阶段的故障主要是由于微机的系统配置文件CONFIG.SYS引发的，而系统配置文件的主要问题是执行了错误的文件，或者是系统配置文件的某行语句格式错误。
    在这一阶段可能出现的故障有：
    1.各种CONFIG.SYS出错信息。
    2.启动后某些设备无法使用。
    3.压缩盘丢失，硬盘容量大大减少。
    4.电脑死机，进入死循环。
    上述各种故障的处理方法如下：
    1.各种CONFIG.SYS出错信息的原因及处理方法
    CONFIG.SYS出错，在出错信息中常常会指出CONFIG.SYS文件中出错的具体语句行，根据提示对该语句行进行仔细的检查，将该行错误的语句删除或修改正确。
    2.启动后某些设备无法使用的原因及处理方法
    某些设备无法使用，主要是由于该设备的驱动程序在系统配置文件CONFIG.SYS中没有加载，或加载错误。如光盘驱动程序加载后，还需要在自动批处理文件中加载硬盘DOS目录下的光盘驱动器访问程序MSCDEX.EXE，否则光盘将不能使用。当然，虽然某些设备无法使用，但通常不会影响微机启动。
    3.压缩盘丢失的原因及处理方法
    硬盘经过压缩之后，微机启动时将在系统配置文件CONFIG.SYS中加载压缩盘的管理文件DRVSPACE.SYS，如果微机启动后硬盘变为几M，请查看配置文件中是否有加载DRVSPACE.SYS的语句，如果没有，可以添上“DVICEHIGH=C:＼DOS＼DRVSPACE.SYS/MOVE”。
    4.电脑死机的原因及处理方法
    在这一阶段出现的电脑死机，与前面的几个阶段的死机现象相比，这种死机之前微机将有“Starting MS-DOS…”的提示，说明已经进行了DOS的引导过程。出现死机的原因主要是系统配置文件CONFIG.SYS加载了错误的命令，或执行了错误的文件。
    为了准确判断问题出在系统配置文件CONFIG.SYS的哪一行，可以在微机启动出现“Starting MS-DOS…”时按住“F8”键，然后一行一行地去执行系统配置文件，当执行到某行时死机，说明该行配置命令发生问题，应当仔细检查并修正。
#1    九、第七阶段可能出现的各种故障
    在这一阶段，系统将装载DOS的命令管理程序COMMAN.COM，以便对系统的各种操作命令进行协调管理，这一阶段可能会出现的故障提示是： “Bad or missing command interpreter”。出现这种故障主要有以下三种原因：
    1.硬盘根目录下没有COMMAND.COM文件。
    2.硬盘要目录下的COMMAND.COM文件被破坏。
    3.COMMAND.COM文件与两个系统隐含文件不属于同一个版本。
    解决上述问题的方法是将与系统隐含文件版本相同的COMMAND.COM文件拷贝到硬盘的根目录下，覆盖原来的文件。在DOS4.0以上的版本，也可以用系统传送命令SYS处理。
#1    十、第八阶段可能出现的各种故障
    如果硬盘根目录下有自动批处理文件（AUTOEXEC.BAT），微机将在这一阶段读取自动批处理文件AUTOEXEC.BAT，并一步一步地执行批处理文件中的各条命令。
    这一阶段可能出现的故障与微机在执行系统配置文件时出现的故障类似，常见的故障现象有：
    1.各种AUTOEXEC.BAT出错信息。
    2.启动后某些设备无法使用。
    3.电脑死机，进入死循环。
    出现上述故障的主要原因有：
    1.自动批处理文件执行了错误的语句。
    2.自动批处理文件中没有执行某些设备的驱动程序。
    3.自动批处理文件调用的文件发生错误。
    处理这些故障的方法与处理因系统配置文件引起的几种故障相似，仍然可以在微机自检后提示“Starting MS-DOS…时按住“F8”键，然后一行一行地执行，最后找出引发故障的语句进行具体的处理。
#1    十一、微机重要资料备份与恢复的方法
    前面介绍的对微机各种启动故障的检修方法中，都没有提到如何利用在故障发生之前系统作的备份信息来检修微机不能启动的故障，事实上，如果在系统能够正常工作的时候，对微机的重要资料进行备份，当系统发生故障时，利用备份信息处理故障，将更加安全可靠。
    1.备份与恢复COMS信息的几种方法
    （1）记录法
    首先开启打印机，装好打印纸，再进入系统CMOS设置程序，选择相应的设置选项，进入具体的设置项目，然后按住键盘的SHIFT键不放，再按键盘的PRINT键，将显示器上的内容打印机纸上，用同样的方法打印其它设置项目内容。恢复时将微机的CMOS设置一项一项地改为打印出来的内容。这种方法虽然繁琐，但却比较可靠，一般用户都能够掌握。
    （2）利用BOOTSAFE备份
    BOOTSAFE.EXE是CPAV软件包中的一个实用程序，利用它可以比较方便地将微机的CMOS设置数据进行备份和恢复。
    备份时将写有BOOTSAFE软件的磁盘插入软驱，键入BOOTSAFE/M命令，再根据提示操作即可。备份完后取出软盘，作好写保护。
    恢复时将记录有CMOS备份信息和BOOTSAFE软件的软盘插入软驱，键入BOOTSAFE/R命令，再按照提示进行操作。
    （3）利用DISKTOOL（6.0版本）或者RESCUE（8.0版本）进行备份
    执行RESCUE后，选择“Create”，再选择软盘驱动器符、软盘类型等后即可将CMOS信息备份到软盘。如果是恢复CMOS信息，应该选择RESCUE的“Restore”，再选择“CMOS Information”。
    （4）利用DOS的DEBUG进行备份
    DOS的调试程序DEBUG也可以备份CMOS资料，但这种备份方法不适合普通用户，这里不作详细介绍。
    2.备份与恢复硬盘主引导扇区的几种方法
    （1）用KV300进行备份
    KV300杀毒软件在处理硬盘主引导记录故障方面有很好的效果，同时也可对硬盘主引导扇区进行方便的备份。
    用KV300备份硬盘主引导信息的命令格式为：KV300/B，这种格式向A盘备份一个是无病毒的硬盘主引导信息档案，其名为：HDPT.DAT。然后请在软盘的标签上写明机器型号、硬盘容量、分区大小后长期保存好。
#1    注意：使用这种格式前，应该用KV300软件查C盘证明无病毒再用无病毒的DOS系统软盘引导机器（冷起动机器），以确保内存中无驻留病毒。
    用KV300恢复硬盘主引导信息命令格式为：KV300/HDPT.DAT，这种格式将备份在A盘硬盘主引导信息档案恢复到该硬盘中。
#1    注意：使用这种格式时，不可搞错硬盘!也必须证明这个硬盘以后没重新分区和格式化!
    （2）用RESCUE进行备份
    备份操作与用RESCUE备份CMOS信息相同，恢复时选择RESCUE的“Restore”，再选择“Partion tables”。
    （3）用DOS的调试程序DEBUG备份
    3.备份与恢复硬盘DOS引导记录的几种方法
    （1）用BOOTSAFE备份
    备份与恢复的操作与用BOOTSAFE备份和恢复CMOS设置方法相同，实际上，用BOOTSAFE可以同时对CMOS信息和硬盘DOS引导记录（BOOT）进行备份。
    （2）用RESCUE进行备份
    备份的方法与用RESCUE对CMOS备份的方法相同，恢复时选择RESCUE的“Restore”再选择“Boot records”。
    （3）用DOS的调试程序DEBUG备份
#1    十二、一种全能备份软件
    全能备份软件非常适合普通的电脑用户，该软件可以对CMOS设置信息、硬盘主引导记录（含分区表）、DOS引导记录、系统配置文件、自动批处理文件及硬盘根目录下的一些驱动软件进行一次性的备份和恢复，也可以只对其中的某个具体项目进行备份或恢复。
    一次对系统的所有重要资料进行备份的方法是：将该备份软盘插入软驱，用软盘启动微机，等到出现提示信息后键入SYSBAK，全部的重要资料就可以一次备份完成。
    一次对微机所有重要资料进行恢复的方法是：将该备份软件插入软驱，用软盘启动微机，等到出现提示信息后键入SYSRES，全部重要资料就可以得到恢复。
    至于用这张软盘对各种具体资料的单独备份的方法，请阅读软盘中的说明。
#1    注意：经常用这张软盘对系统进备份，尤其是在下列各种情况下更应该重新备份：
    1.CMOS设置被更改后。
    2.硬盘被重新分区或格式化后。
    3.系统配置文件被更改后。
    4.系统自动批处理文件被更改后。
    5.硬盘被更换后。
    还应当注意的是，一台电脑只能使用一张软盘，千万不能将软盘搞错，否则将造成不良后果。用软盘对系统资料进行备份后，应当将软盘进行写保护，并注意备份微机的型号、微机机主、备份时间等。
    利用这张专用的备份盘，可以修复微机百分之七十的各种不能启动的软故障。如果使用了这张软盘，并且时常对微机进行了备份，当你的微机出现了不能启动的故障时，建议你先用这张软盘进行进行，大多数情况下都能排除故障。
