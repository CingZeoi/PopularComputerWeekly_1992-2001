#1浏览加速器——百试不爽的软道理
　　同Internet的连接无论是利用56K的调制解调器、 还是借助综合业务数据网（ISDN） ，或是通过直接的网络连接，浏览Web的速度都可能不够快。来自Internet的数据毕竟不能以高于调制解调器和电话线路（或者网络连接线路）发送能力的速度进入计算机。改造线路扩大带宽这种硬道理谁都懂，但是它决非一朝一夕就能奏效的; 要想立竿见影地提高浏览速度，除了使用浏览加速器以外似乎别无它法。
　　浏览加速器就是利用现有连接使网页在浏览器窗口中显示得更快的实用程序。首次浏览某一网页后，即将该网页下载到用户本地硬盘上，这样，当用户再次浏览网页时，实际上就是从本地硬盘上调用它，该网页就可以立即出现在浏览器的窗口中。
　　几乎所有的浏览加速器都是将自己安装成本地代理服务器来同浏览器通讯的。
　　用未加速的浏览器阅读网页内容时，调制解调器是闲着的（数据传输指示灯不闪亮）。当单击新的链接向Internet请求数据时，浏览器才从网络中下载相应数据，这时你不得不等待。
　　用已加速的浏览器阅读网页内容时，调制解调器始终处于工作状态，表现为数据传输指示灯始终闪亮。在你阅读网页上某些内容的时候，调制解调器同时在下载该网页链接的内容，所以当单击网页上的某个链接时，相应的内容很可能早就已经下载到硬盘中了。
　　浏览加速器能够加速对某些网页而不是所有网页的访问速度，它不能缩短下载首次访问网页的时间。如果迅速地在网页之间跳来跳去，就得不到明显的加速效果。
　　只要网络主要还是通过具有带宽限制的通讯线路来构造，浏览的速度屏障就永远存在，浏览加速器也就永远不会“失业”。浏览得越多，浏览加速器的作用就越大。但是所获得的浏览速度的提高和浏览加速器带来的操作复杂性相比是否值得，只有用户才能作出公正的评价，因此，这里面就有一个选择问题。只有那些简洁易用、对系统原来的设置影响最小甚至根本没有影响的加速程序才是普通用户的首选程序。

#1　　[功能概览]
　　如果不打算升级古老的慢速调制解调器，但是却希望大幅度提高浏览的速度，那么Go ahead Got it!（以下简称GG）是最佳选择。使用GG，可以通过指定预取哪些网页和何时需要预取这些网页来随心所欲地控制浏览加速过程。GG能预取FVP （Frequently Visited Pages，频繁访问网页） 和包含在当前浏览网页中的链接指向的网页内容，而且在选定的远程网页内容发生变化时还能够及时给出通知。它能够以一定的间隔定期自动下载指定的站点。在支持离线浏览方面，GG允许用户通过已经预取来的网页链接列表来浏览预取的网页内容。
#1　　[工作描述]
　　GG是作为代理服务器工作的。能够随系统自动启动为任务条上系统托盘区域的图标 ，通过该图标右键菜单中的Remote Control指令，可调出^211501a^所示的类似遥控器的控制板。
　　控制板上设置了6个按钮分别用于访问公司主页、 显示缓冲区中文件内容、开始下载指定网页、打开预取文件列表等功能。单击打开的预取文件列表中的相应链接即可将所链接的文件显示在浏览器窗口中。采用单独的页面来显示请求预取的网页中内容有变化的网页列表，通过右键菜单中的Add site指令或者控制板上的同名按钮，可以向请求预取的网页列表中添加新的要预取的网页链接。
#1　　[综合评价]
　　实际使用表明，在许多站点的浏览过程中，GG都显示出了十分明显的加速效果。当然，有时GG也糊涂，以致无法取得某些网页内容，而不得不将其重新启动，此时同GG配合的浏览器依然能够照常使用，因为重新引导的时候，它能够自动恢复原来的代理设置。
　　GG可能是目前功能最丰富完美、加速效果最明显的浏览加速器和离线浏览器。只是工作起来有点儿过于殷勤，例如总是随系统自动启动且显示一个网页内容。GG最大的缺点是性能不够稳定且使用方法难掌握。Go ahead Got it!对于初级用户是个小小的挑战，但是对于网络高手而言则是浏览加速程序中的佼佼者。

#1　　[功能概览]
　　NetSonic（以下简称NS）将自己安装为本地代理服务器，将用户浏览过的内容保存到自己设置在硬盘上的缓冲区里。以后再次浏览时，浏览器就直接到缓冲区中请求所需要的数据。NS提供控制板供用户进行相应的操作。通过控制板上的相应按钮可以将控制板缩小或者隐藏起来以便不影响浏览。NS具有可设置的智能缓冲功能，有Stabdard Acceleration（标准加速）和Maximum Acceleration（最大加速）两种设置选择。
　　NS的预取功能有Pre－fetch all links（预先取得所有链接的内容） 和Pre－fetch visited links only（只预取访问过的链接）两种。无论选择哪种，NS只预取已经访问过页面上的链接指向的内容，而不预取其它页面中的链接指向的内容以获得最大的性能。NS允许用户根据硬盘空间情况自己指定缓冲区大小，它最多可以使用100M硬盘空间来缓冲网页内容。
　　选择标准加速时， NS将对缓冲区中Web页面文本和图形内容同网络上目前的内容进行较频繁的对比，在没有更新的情况下直接将缓冲区里的相应内容提供给浏览器，进而实现加速作用。而一旦发现内容有所更新就继续下载更新了的内容。选择“最大加速”时，NS就降低了对Web页的检查频度，从而使加速效果明显提高。
#1　　[工作描述]
　　NS能够自动随浏览器一起启动。启动后在任务条系统托盘区域放置图标并在屏幕上放置控制板，见^211501b^。
　　第一次浏览某个网页的时候，基本上没有加速效果。以后再浏览同样内容的时候，浏览器将从NS的缓冲区中取得需要的数据，而NS就在后台检查当前浏览的网页在网络上的内容，一旦发现更新就发出声音告诉用户，这时就需要单击浏览器工具条中的刷新或者Reload按钮来显示更新后的网页内容。
　　在不使用NS的情况下，首次下载某个网页，差不多用了30秒的时间，再次下载同样的网页需要的时间也将近20秒;使用NS以后，下载同样的网页，带有更新内容时需要大约8秒钟，而完全从缓冲区中取得不带更新内容的网页拷贝则仅需要大约3秒钟。
　　NS在下载升级了的网页内容时，采用了‘选择更新’的策略，这样就减少了往返于Internet的时间。^211501c^反应了这种减少的程度。
　　从图中可以看出，没有使用NS时，完成某项任务往返了20多次，而采用了NS以后，完成同样的任务往返次数不到10次。
#1　　[综合评价]
　　NetSonic是我使用最早、使用时间最长的浏览加速器程序。其出品人立志将NetSonic做成世界上最好的浏览加速器。而NetSonic在实际的工作中表现也相当好，简单易用，对浏览器设置没有任何影响。另外，NetSonic几乎支持所有的主流浏览器。和Go ahead Got it!是熟练用户的最佳选择相反，NetSonic是初级用户的最佳选择。

#1　　[功能概览]
　　WebEarly（以下简称WE）具有不少独特的功能，例如能够使用单独的窗口按访问时间顺序列出已访问过的Web页面中发现的全部E－mail地址；能够过滤掉许多原本显示在浏览器窗口中的广告信息，能够将预取文件限制为某种特殊的类型 （例如限制WE只预取声音文件而不预取图像）等等。不过WE的其它功能远远比不上GG，例如它不像GG那样允许用户最大限度地决定要预取的网页。和大多数其它同类程序相似的是，WE对网页内容进行缓冲并预取包含在当前正在浏览的网页中的链接指向的网页内容。它最叫人喜欢的特点是在需要返回访问过的网页时，只要单击浏览器地址栏，而后键入相应网页名称中包含的任意几个字符，就会弹出一个单独的窗口显示包含相应字符串的网页名称列表，这时只要单击其中的一个即可打开相应的网页。

#1　　[综合评价]

　　Speed Surfer（以下简称SS）是一个功能比较单一、不具有离线浏览和缓冲区管理功能的浏览加速程序。不过，在预取与当前网页有链接关系的网页内容上，它工作得颇为出色。由于将自己安装成了代理服务器，所以它使用起来要麻烦一些。

#1　　[功能概览]
　　NetAccelerator（以下简称NA）是目前最简单、问题最少的浏览加速器程序， 也是唯一跳出将自己安装为代理服务器以及需要用户通过选项菜单来设置这个俗套的程序。安装NA以后，用户的浏览操作和没安装时一样，唯一的变化就是速度加快了。它主要在后台预取和当前网页有链接关系的其它网页及用户经常访问网页（FVP） 的内容。和GG比较起来，它缺少某些诸如离线浏览和控制方面的功能。
#1　　[工作描述]
　　启动以后即打开一个包含有几个按钮的控制板，这些控制板是用来启动IE或者NN浏览器，见^211501e^。
　　事实上，在还没有单击启动按钮之前，NA就开始“暗地里”运行浏览器程序（姑且称为后台浏览），单击浏览器启动按钮选定浏览器后，就再运行一个“公开的”浏览器程序（可称为前台浏览）。在进行前台浏览的同时， 后台浏览就预取和当前网页有链接关系的网页以及FVQ网页的内容，并将其保存在浏览器的缓冲区中以供前台浏览，进而提高浏览速度。当退出NA同时关闭前台浏览器后，会看到调制解调器依旧在工作。这就是NA的奇妙之处。它的另外一个与众不同之处在于不以代理服务器姿态出现，所以不需改变浏览器的任何已有的设置。
　　在NA的控制板上有一个Pause按钮，它可以用来暂停网页的预取工作，以便利用整个连接的带宽资源来处理E－mail收发或者其它非浏览性网络事务。使用中体会到，NA唯一的问题就是显示在屏幕上的控制板在某种程度上影响浏览，万幸的是可以将其最小化到任务条上。
#1　　[综合评价]
　　如果追求实用和朴实无华，NetAccelerator是一个较好的选择。它功能专一，简单易用，是初级用户仅次于NetSonic的选择。

#1　　[功能概览]
　　Java Booster 1.0a（以下简称JB）和其他浏览加速器不同，仅仅加速已经访问过的网页，它使用经过智能化设计的可搜索缓冲系统来取代浏览器的缓冲作用。用作离线浏览器的时候，JB肯定是最好的选择，因为它不仅能够无限制地在其缓冲系统中保存指定的网页，而且还能够保存所有连接到这些网页的Java Applets，这样用户就不用下载这些Java Applets。
　　JB的出众之处是其智能检索功能。利用内置的文本检索功能，它可以列出缓冲区中包含指定文本的网页，单击某个链接即可将相应的网页内容显示在浏览器窗口中。
#1　　[工作描述]
　　使用JB的前提是改变浏览器的代理设置，以便让浏览器从JB取得网页数据而不再直接访问Internet。此后就只有先运行JB，而后才能通过它来访问Internet。JB通过一个小型控制板窗口中的按钮在离线和在线状态之间切换，也可通过按钮来打开显示在浏览器窗口中的选项设置窗口。
#1　　[综合评价]
　　它仅仅是通过缓冲Java Applets来节省在线时间的加速程序，远远不能满足复杂的浏览加速需要。突出之处是离线浏览功能。
