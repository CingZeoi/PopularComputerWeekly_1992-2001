Win 98安装问与答

龚勇
1998年 第29期 10版

#1　　这个并不太热的夏季因为Win98的到来而变得热烈起来，面对这个庞大的操作系统，我们都跃跃欲试。也许你就正忙着让自己的PC感受Win98的魅力，且慢，在你行动之前，请看一看——
  　问：据说WIN 98可以从DOS方式下直接安装，也可以从WIN 95的资源管理器中直接升级，那么这两种方法究竟有多大区别?如果我在升级WIN 98时不想覆盖原来的WIN 95目录，那么又该如何操作?
#1　  答：从MS DOS 6.x下进行WIN 98的安装与从WIN 95的资源管理器中升级区别并不大，如果将WIN 98安装在原WIN 95目录中，那么升级后就可以保留原WIN 95系统下所有软、硬件设置，沿用原WIN 95系统下已安装的所有应用软件。但在DOS方式下直接安装WIN 98时，安装程序将提示用户确定安装路径和目录名；而从WIN 95资源管理器中直接运行WIN 98的Setup文件升级时，安装程序自动将WIN 98文件安装在原WIN 95的路径和目录中。因此，如果我们希望在安装WIN 98后仍然保留原WIN 95的路径和目录名而不和新安装的WIN 98发生任何联系，那么可以从DOS状态下重新安装WIN 98，在提示安装路径时另外指定盘符和目录，但这样就必须为WIN 98安装所有器件的驱动程序，同时安装所有仍将使用的应用软件。注意，采用这种方法安装WIN 98时应该保留原WIN 95的启动盘，以后在手工卸载WIN 98时有用。
　　问：安装WIN 98对电脑硬件配置有什么要求?
#1　　答：显然WIN 98的最低配置要求只是486DX－66MHz电脑加16MB内存，但根据实际运行情况，应使用如下配置才能得到较满意的运行效果：CPU在200MMX以上，内存至少为32MB，48MB或以上更好；虽然一般VCG显示卡符合WIN 98最低要求，但要充分发挥WIN 98的性能，最好还是具有3D加速器的显示卡。
　　问：当安装或升级成WIN 98后，是否系统就具备了FAT32模式管理能力?
#1　　答：从WIN 95或WIN 95 OSR2中升级成WIN 98后，系统并不一定都具备FAT32模式，而是沿用原来在WIN 95或WIN 95 OSR2中已经使用的文件管理模式，例如从WIN 95中升级的沿用FAT16，从WIN 95 OSR2中升级的则沿用原来的FAT16或FAT32。
#1　　另外，如果在升级成WIN 98以后仍然想使用双操作系统（WIN 98 ＆MS DOS 6.x或者WIN 98 ＆Windows NT），那么就不能使用FAT32模式管理硬盘，因为MS DOS 6.x和Windows NT不能访问使用FAT32管理的分区。
#1　　对于希望将硬盘转换为FAT32模式的用户，微软公司在WIN 98中专门提供了FAT16至FAT32的磁盘转换程序，这样即使已经安装好的系统也可以方便地将硬盘文件系统转换成FAT32模式。
　　问：在安装WIN 98后，如何才能设置一个丰富多彩的桌面主题。
#1　　答：在WIN 98之前，我们已经可以从IE4预览到WIN 98的桌面风格，可要安装丰富多彩的桌面主题，还得自己动手：
#1　　1.打开“控制面板”，单击“添加或删除程序”。
#1　　2.选择“Windows安装程序”选项板，系统会在“组件”窗口中显示出经过搜索后确定的所有可安装的Windows组件，这时选择“桌面主题”，然后插入WIN 98光盘，单击“从磁盘安装”，启动安装。你还可通过选择“详细资料”了解你所安装的组件的详细构成。
#1　　3.安装完毕，“控制面板”中会出现一个代表桌面主题的新图标，单击该图标，在弹出的“桌面主题”窗口中你可以为WIN 98设计一种自己喜爱的主题，也可以自由地制定有关桌面主题的各个属性。
　　问：当系统升级成WIN 98以后，原先在WIN 95中正常运行的应用软件也还能正常运行吗？
#1　　答：一般来说，原先可以在WIN 95中正常运行的软件都可以自动适应WIN 98环境正常运行，比如微软的Office 95/97等、数据库管理等常规软件和大部分游戏软件。但情况并不都是如此，例如中文之星V 2.97就必须加补丁。如果你所使用的某种软件在升级WIN 98后不能正常运行了，那么你通过与软件供应商联系后可以得到指导或针对该应用软件的补丁程序。像中文之星V 2.97就将在WIN 98中运行时需要的补丁程序放在因特网上，供用户免费下载。由于目前的WIN 98还存在些BUG，所以一旦我们升级成WIN 98后，碰到某个软件不能运行，那么就应该首先查清是否真是与WIN 98不兼容还是因为使用的软件本身没有正确设置好，然后再去寻找补丁程序或采取其它措施。
　　问：我在安装WIN 98时，总是出现一些莫名其妙的错误信息，究竟是怎么回事?
#1　　答：对于在安装过程遇到的一些常见错误信息，我们提供以下的意见和解决方法：
#1　　1.如果你在安装WIN 98的过程中出现SU0167的错误信息，那是因为当前目录中存在名为“Desktop”的目录，而WIN 98将生成一个名为“Desktop”的目录存放系统文件，因此安装程序不能将WIN 98安装在这个目录中。解决的方法是：将当前目录中的“Desktop”目录删除或重新命名，然后再运行安装程序。
#1　　2.如果安装过程中出现SU－0011的错误信息，那是因为你对硬盘设置了口令，使得安装程序不能完成安装，解决办法是删除口令设置。
#1　　3.当你屏幕显示有如下错误信息的某条时：Standard Mode:Invalid DPMI return”，“Standard Mode:Fault in MS－DOS Extender”，“Standard Mode:Stack Overflow”，意味着你必须将Config.exe文件中的内存管理器删除掉。
　　问：如果WIN 98是通过从WIN 95资源管理器直接运行Setup进行升级安装的，该如何才能安全卸载它呢?
#1　　答：对于这种情况，可采用自动卸载WIN 98，恢复WIN 95的方法。
#1　　这是对于直接从WIN 95中升级WIN 98的系统处理。操作步骤是：“开始”→“设置”→“控制板”→“添加/删除程序”→“卸载Windows 98”。
#1　　当系统开始卸载时的屏幕提示过程是：
#1　　首先提示“在升级WIN 98后压缩过硬盘或进行FAT32转换的，请不要卸载WIN 98” ，此时如果确认卸载选“Y”继续。
#1　　再次提示“卸载前系统将检查硬盘”，让你再次确认，如果选“Y”继续。
#1  　系统开始使用scandisk程序扫描硬盘，提示准备卸载、关闭WIN 98。提示继续自动卸载选择“Y”，如果选“Y”，系统开始自动卸载WIN 98，屏幕开始显示卸载过程，如恢复硬盘分区表、主分区表；恢复目录、检查文件和目录等，最后显示为“反安装程序将重新启动（WIN 95）系统，如回车后不能重新启动，可关闭电源后再次启动”的提示。通常回车后都能重新启动系统，进入WIN 95，此时WIN 98的卸载正常完成，整个过程约需2～5分钟。
　　问：若是通过在硬盘上单独建立WIN 98目录并重新安装所有硬件驱动程序和应用软件的方式安装的WIN 98，要想卸载它，该怎么办呢?
#1　　答：由于不是通过WIN 95升级，而是另外安装的WIN 98，如果硬盘上仍然完好保留原WIN 95的目录，可采用手工卸载WIN 98，恢复WIN 95的方法：
#1　　进入CMOS设置，将BOOT顺序改为A，C。
#1　　将原WIN 95启动盘放入A盘，启动系统，当屏幕出现A：\>提示符后，执行“A：\>SYS C:”命令。
#1　　当提示“System transferred”时，取出软盘，重新启动系统。
#1　　系统启动时，提示出现“Starting Windows 95……”然后会直接进入DOS的C盘提示符下，此时需重新修改Windows定制启动文件“msdos.sys”。方法：先取消msdos.sys的系统、隐形和只读属性。然后执行“C:\>edit msdos.sys”命令，开始修改文件内容。检查“[paths]”项中“WinDir=盘符\WIN 98目录”、“WinBootDir=盘符\WIN 98目录”和“HostWinBootDrv=C”这三句，并将语句中的WIN 98的安装位置和目录名改为WIN 95的安装路径、目录名和WIN 95安装目录所在盘符就可以了。
#1　　修改结束后（注意不要删除或改动其它任何无关的字符）存盘退出edit程序，然后执行“attrib msdos.sys＋s ＋h ＋r”命令恢复msdos.sys文件原属性。
#1　　修改“msdos.sys”文件后，还可以根据实际情况对“autoexec.bat”文件中相应WIN 95的路径进行必要的修改。
#1　　以上修改结束后，可以将CMOS设置中的BOOT顺序再恢复成C，A等。然后重新启动系统就可以进入WIN 95。此后如果不再需要WIN 98就可以将WIN 98的目录删除，如果保留WIN 98目录，那么以后就可能采用像手工恢复WIN 95的方法恢复WIN 98。
#1　　（以上方法不适于FAT32分区的硬盘）
