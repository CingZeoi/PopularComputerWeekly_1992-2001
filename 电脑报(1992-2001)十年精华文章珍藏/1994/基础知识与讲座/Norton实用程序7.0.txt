Norton实用程序7.0


1994-01-07

        1.将装有Norton系统程序的磁盘放入软驱中，然后打入：
        a:install(或b:install)
        这将启动Norton实用程序的安装程序。在安装过程中，可以用键盘来选择屏幕上的选项，也可以用鼠标来选择。若用光标键或Tab键来选择，选中之后再用空格或回车键确认。
        2.选择显示器
        第一屏让你设置显示器。可以是彩色显示器、单色显示器或液晶显示器。
        3.重要信息
        第二屏显示提示信息，如果当前硬盘上包含了你想恢复的被删除文件，或者你想恢复因故格式化了的磁盘，安装程序会提示你终止安装。因为Norton实用程序可能会覆盖这些被删除的文件。
        4.安装
        第三屏显示：
        a)安装程序从安装盘拷贝Norton实用程序到你选择的硬盘目录中。
        b)安装程序允许你修改CONFIG.SYS和AUTOEXEC.BAT文件，使得它们完全适合Norton实用程序。
        c)你也有机会配置Norton实用程序，以便在你的系统中更好地运行。
        5.用户名和公司名
        第四屏请求输入用户姓名和公司名称。
        6.安装方式
        第五屏选择安装方式：
        a)全部安装：安装全部产品并且自动配置你的系统(需3.68MB磁盘空间)。
        b)客户安装：允许你选择要安装的内容和按你的要求配置系统。
        7.指定安装目录
        第六屏提示输入安装Norton实用程序的目录，缺省目录是C：\NU。用户可以输入自己的目录名。
        如果你的硬盘上有一个更早的Norton实用程序的版本，并且你没有为新版本指定一个不同的目录，那么安装程序在安装新版本之前将删去旧版本。
        8.选择程序文件
        如果是选择全部安装，就直接进入安装菜单，否则将出现选择屏幕。在这一屏，你可以选择要安装的程序，以便磁盘有足够的空间来存放它们。或者选择另一个驱动器(该驱动器有足够的空间)，或者退出安装程序。
        在屏幕中显示的程序只是一部分，通过右边的标柱杆可以显示后面的程序。当磁盘空间不够时，可以只选择用户感兴趣的部分。
        如果不选择，则缺省值是全部选择。即安装程序自动选择所有的Norton实用程序进行安装(除非你的硬盘上没有足够的空间)。
        当你选择或取消选择要安装的程序时，磁盘上可用的空间，你所选择的要安装的文件所需的空间会发生变化。
        9.选择配置选项
        在选择完要安装的程序之后，将出现选择配置屏幕：
        ┌────────────────────┐
        │             选择配置选项               │
        │ 当你已经选择了所有你要安装的文件之后， │
        │ 你可以照你喜欢的方式配置Norton实用程序 │
        │               配置选项                 │
        │ Run Norton Disk Doctor on startup      │
        │ Run IMAGE on startup                   │
        │ Load SmartCan on startup               │
        │ Install Norton Cache                   │
        │ Add the NU directory to the PATH       │
        │ Add the SYMANTEC environment  variable │
        │ * Configure Norton Utilities           │
        │ Create Windows Group for the Utilities │
        │ Continue   Return to DOS               │
        └────────────────────┘
        你可以选择要配置的项，使Norton实用程序更适合你的使用习惯和要求。在屏幕中显示的选项只是一部分，通过右边的标柱杆可以显示后面的选项。
        你可以用鼠标或空格键选择或取消任何一个配置项。前面有“*”的项是不可取消的。
        10.开始安装
        当前面的选择完成后，Norton实用程序的安装工作便开始进行。在安装过程中，有相应的提示信息，告知用户什么时候换盘以及显示正在拷贝的文件名等。
        在安装完成之后，还要存放系统文件，改换AUTOEXEC.BAT和CONFIG.SYS文件内容等屏幕提示，通常使用系统缺省值，直接敲回车即可。
        二、Norton实用程序的启动和操作
        1.Norton实用程序的启动
        在DOS提示下，键入命令：
        C〉NORTON
        NORTON.EXE程序是一个管理程序，给出一个调用其它Norton实用程序的工作窗口。在窗口左边的命令(Commands)栏中，列出了各种Norton实用程序的名称。在窗口右边的描述(Description)栏中，给出了命令栏中当前命令的简要说明、选择项和参数等内容。在左边的命令栏用光标键上下移动时，右边描述栏的内容也相应更新。
        屏幕顶行的主菜单包括：
        A)菜单(Menu)
        B)配置(Configure)
        C)帮助(Help)
        窗口中的当前命令，会自动出现在屏幕底部的Norton命令行上。如果要输入命令参数，可在命令行的命令名之后输入。按下Enter键或者连击鼠标按钮，则会运行命令行中的程序。
        2.Norton实用程序的操作
        ①下拉式菜单：
        屏幕的顶行是菜单长条，它是下拉菜单的主菜单，通过它可以访问下拉式菜单。主菜单上显示当前下拉式菜单的标题。
        ②用键盘操作下拉式菜单
        欲选择下拉式菜单上的命令，可进行以下操作：
        ·按下Alt键或F10键打开下拉菜单，用→、←键在主菜单间移动。
        ·按Alt键和主菜单标题的第一个字母可以下拉出此菜单。
        ·用↑、↓键移动到所期望执行的命令上。
        ·按Enter键执行此命令。
        ·使用热键。每个菜单命令名上都有一个以不同颜色显示的字母，按下该字母也可 选择此命令。
        ·用Esc键可退出一个菜单。
        ③用鼠标操作下拉式菜单
        ·将鼠标指示器定位到主菜单名上，然后按鼠标按钮就可以下拉出此菜单。
        ·从菜单上选择菜单条目，在该条目上按鼠标按钮即可执行它。
        ·在菜单之外的空白区域中按下鼠标按钮，可关闭该下拉菜单。
        三、Norton实用程序介绍
        Norton实用程序是一个功能十分强大的工具程序组，它能够进行数据恢复、磁盘修补、目录排序、修复分区信息及被破坏了的硬盘数据等。
        Norton实用程序按其功能大至可分为四个部分：
        a)RECOVERY    文件恢复和数据恢复组。还包括维护数据的安全性、诊断用户磁盘、对用户磁盘进行低层编辑等实用程序。
        b)SECURITY    数据安全性组，可防止数据的意外丢失，并允许用户限制或者修改数据的存取权限，达到保密的目的。
        c)SPEED       系统性能组。通过测试各种硬件特性和改进数据在用户磁盘上的组织方式，提高系统的速度和效率。
        d)TOOLS       工具程序组，有的增强了系统功能，有的是系统没有的功能。每个Norton实用程序都有其独特的功能，下面根据功能分别介绍它们。
        (一)、文件恢复和数据恢复组(RECOVERY)
        Norton实用程序7.0的文件恢复和数据恢复组由10个独立的程序组成，它又分成四个不同的部分。
        a)恢复被删除的文件及修复有故障的文件。
        b)恢复被格式化了的磁盘。
        c)保护磁盘以防止数据丢失。
        d)诊断计算机硬件问题。
        1.磁盘诊断(Norton Disk Diagnostics)
        磁盘诊断命令由独立程序NDIAGS.EXE完成。它可以用于交互测试、内存测试、打印指定文件的报告等。
        命令格式：
        NDIAGS/AUTO:n［REP:file］
        NDIAGS/SPEC:file1［/REP:file2］
        NDIAGS/RESET
        NDIAGE/COMP
        NDIAGS/BURNIN:x［/SPEC:file1］［REP:file2］
        /AUTO:n       自动进行非交互测试的时间周期。(在两屏之间延迟n秒)
        /REP:file     打印一个指定文件的报告。
        /SPEC:file    用于设置要打印报告的文件集合。
        /COMP         清除系统，以便运行内存综合测试。
        /RESET        重新设置存储在SYMCFG.BIN文件中的配置。(如果计算机任何硬件发生变化时，需要做这项工作)
        /BURNIN:x     自动进行非交互测试x次。(如果命令行中包括/SPEC选项，那么指定的非交互测试将被执行)。
        2.磁盘医生(Norton Disk Doctor)
        磁盘医生实用程序(NDD.EXE)是Norton实用程序中最重要的一个程序，它可以用于诊断及修复磁盘，进行多种测试，查找分区表、DOS引导记录、文件分配表、文件目录及文件结构等。如果在测试或诊断的过程中发现错误，还可以依据有关提示信息而加以改正。
        在遇到如下情况之一时，可使用磁盘医生命令：
        a)需要诊断磁盘，以测试各种可能发生的磁盘故障。
        b)由可能是引导盘或分区有错而导致的系统不能正常启动。
        c)当目录、文件无故丢失。
        d)磁盘工作不正常。
        e)期望用磁盘医生进行系统分析或研究。
        命令格式：
        NDD ［d:］［d:］…［/C|/Q|/DT］［/R［A］:pathname］［/X:drives］
        ［/FIXSPACES］［/SKIPHIGH］
        NDD ［d:］［d:］…/REBUILD［/SKIPHIGH］
        NDD ［d:］［d:］…/UNDELETE［/SKIPHIGH］
        NDD /UNDO［/SKIPHIGH］
        /C                            完全测试，包括表面测试。
        /Q                            除了表面测试之外的全部测试。
        /DT                           仅执行表面测试。
        /R［A］：pathname             写(或附加)报告的文件标识。
        /X：drives                    不测试的驱动器名。
        /FIXSPACES                    修复具有内嵌空格的文件名。
        /REBUILD                      重新建立被破坏了的实体硬盘。
        /UNDELETE                     恢复被先前跳过的一个硬盘分区。
        /UNDO                         将前一次做的修复工作放弃。
        /SKIPHIGH                     跳过正在使用的高端内存。
        3.磁盘编辑器(Disk Editor)
        磁盘编缉器可以查看并编辑磁盘中的全部内容。磁盘编缉器甚至可以处理DOS所不能识别或访问的磁盘，用户不仅可以用它处理文件和目录，而且还可以处理分区表、引导记录和文件分配表。用户还可以进一步地要求将任意簇号或扇区作为编辑对象，对其进行查看和编辑等操作。所以，用户可以从损坏的簇中恢复单个扇区，找到损伤盘的根目录，或者分析盘中未被使用的部分。用户利用磁盘编缉器还可以查看硬盘中不可见分区内的数据。
        磁盘编辑器还提供了其它许多功能，包括：
        信息功能、查找功能、窗口功能、比较功能、填充功能、剪裁功能、写入功能、恢复功能。磁盘编缉器对于故障磁盘的修复，或者从磁盘中获取某些感兴趣的数据信息，都有独到的功能，是一个十分有用的工具。
        在遇到下列情况之一，可以使用磁盘编辑器：
        a)磁盘医生无法纠正用户所遇到的磁盘问题。
        b)文件中的数据丢失或出现异常，文件分配表或目录中存在问题。
        c)需要从损坏的磁盘中提取数据。
        d)需要在文件中更改数据。
        e)目录内容混乱或不可利用。
        f)查找磁盘中未使用部分的内容。
        g)磁盘上出现未知文件，并且无法正常读写它们。
        h)希望浏览由非DOS操作系统格式化的磁盘。
        命令格式：
        DISKEDIT	［drive:］［path］［filename］［/M］［/X:drives］［/W］［/SKIPHIGH］	
        drive:	用于查看/编辑的驱动器号。	
        path		用于查看/编辑的目录路径。	
        filename	用于查看/编辑的文件名。	
        /M		维护模式：避开DOS，直接查看磁盘。	
        /X		禁止对该驱动器进行绝对扇区处理。	
        drives	不被处理的驱动器列表。	
        /W		使可写入(关闭只读)。	
        /SKIPHIGH	跳过正在使用的高端内存。
        如果命令行中用/M开关，磁盘编辑器将能够访问任何物理硬磁盘的诊断柱面。
        4.磁盘工具(Disk Tools)
        磁盘工具命令用于与磁盘故障有关的数据保护和数据恢复操作。它可以完成：
        a)做一个引导磁盘(Make a Disk Bootable)。
        如果一个磁盘无法作为引导盘，而且DOS的外部命令SYS.COM也无法恢复它，则用该命令可以修复它，使目标盘能够作为引导盘。它还能把DOS的COMMAND.COM文件拷贝到新的可引导磁盘上以及必要时修改硬盘分区表等。
        b)从DOS的恢复中恢复(Recover from DOS's Recover)。
        在用DOS的RECOVER命令恢复某一损坏磁盘时，磁盘中的所有文件都被移至根目录下，且全部是机器生成的文件名。这可能并不能真正满足用户的要求，解决实际遇到的问题。被换名的文件还要进行确认才知道它是什么文件，这其实是非常麻烦和困难的事情。
        该功能则针对由于RECOVER运行带来的混乱而设计的，使用户可以将磁盘恢复到运行DOS的RECOVER命令之前所处的状态。
        c)恢复一个损伤的软盘(Revive a Defective Diskette)
        通过设置新的物理格式化信息来恢复有损伤的软盘，经过这样格式化之后，用户仍然可以访问磁盘上原有的数据。
        d)标记一个簇(Mark a Cluster)
        如果用户认为不应该使用某个簇，这个功能可以对该簇作“坏”(Bad)标志，这样标志后可以防止DOS使用它。而对簇作“好”(Good)标志，则是允许DOS使用它。
        命令格式：
        DISKTOOL ［/MAKEBOOT］［/SKIPHIGH］
        DISKTOOL ［/DOSRECOVER］［/SKIPHIGH］
        DISKTOOL ［/REVIVE］［/SKIPHIGH］
        DISKTOOL ［/MARKCLUSTER］［/SKIPHIGH］
        /MAKEBOOT                做一个引导磁盘表
        /DOSRECOVER              从DOS的恢复中恢复
        /REVIVE                  恢复一个损伤的软盘
        /MARKCLUSTER             标记一个簇
        /SKIPHIGH                跳过正在使用的高端内存
        5.文件修复(File Fix)
        文件修复命令可诊断出Lotus1-2-3、Symphony、dBASE等的数据文件被损坏后出现的问题。
        文件修复命令检查受损文件，并从原始文件中恢复尽能多的数据，自动重建一个无错的文件副本。多数文件可由文件修复命令自动修复，但有的被损坏的dBASE文件缺省完整文件头，这时你可能得动手建立一个与该文件中的数据相符的文件头。修补dBASE的文件头或许是所有文件修补中最难的工作。
        文件修复命令还可以“UnZap”dBASE文件。dBASE的Zap指令将使文件的数据区收缩为仅剩文件头。如果对原数据文件在作Zap之前忘记了备份，将使数据无法恢复，这是非常糟糕的事情。文件恢复命令的UnZap功能能够发现文件的数据并自动恢复它们。
        命令格式：
        FILEFIX［pathname］
        pathname  要修复的文件名。文件恢复命令根据文件扩展名确定目标文件的类型。
        6.映像(Image)
        映像命令有助于被意外格式化了的磁盘的恢复。实际上映像命令是将系统中的重要数据以文件的形式保存下来，要保存的信息包括：引导记录、文件分配表、根目录区等，将这些数据存放在文件IMAGE.DAT中，IMAGE.DAT中的数据是最后一次执行映象命令的数据，它能恢复的文件也是以此为准，因此，更新IMAGE.DAT数据是非常重要的。
        IMAGE.DAT文件必须放在根目录中才有效，创建时，自动被置为只读方式。
        命令格式：
        IMAGE［drive］［/NOBACKUP］
        drive        保存信息的驱动器。
        /NOBACKUP    不创建信息文件的备份文件
        7.营救(Rescue)
        营救命令由独立的执行程序RESCUE.EXE完成。它用于创建一个营救磁盘或从一个营救磁盘恢复。在创建营救磁盘时，它把下列系统的信息存入软盘中：分区表、引导记录、CMOS值，而今后一旦系统有问题，则可以用营救盘中的信息来恢复系统。
        命令格式：
        RESCUE   ［/CREATE［:location］］［/SKIPHIGH］
        RESCUE［/RESTORE［:location］］［/SKIPHIGH］
        /CREATE        创建一个营救磁盘
        /RESTORE       从一个营救磁盘中恢复
        /location      存放营救文件的驱动器或目录
        /SKIPHIGH      跳过正在使用的高端内存。
        8.删除延迟(SmartCan)
        我们知道，当文件被删除时，除了要去掉目录之外，还要释放它占用的磁盘空间，即要释放它占用的FAT区中的链表。而这些刚释放的链表立即就可以被其它文件所占用。如果遇到这种情况，以后对占用此区的被删除文件的恢复就会遇到困难，甚至变得不可能。删除延迟的功能是使文件被删除之后，它释放出来的链表(也意味着它原来占用的磁盘空间)，在一定的时间周期内不被分配给别的文件使用。时间周期为5天。
        命令格式：
        SMARTCAN /STATUS
        SMARTCAN /ON［/SKIPHIGH］
        SMARTCAN /OFF NOINSTALL
        SMARTCAN /CONVERT
        /STATUS            显示删除延迟状态
        /ON                使删除延迟工作
        /OFF               使删除延迟不工作
        /NOINSTALL         不安装删除延迟(与/OFF功能一样)
        /SKIPHIGH          跳过正在使用的高端内存。
        /CONVERT           搜索并转换删除保护废弃物
        删除延迟的典型状态可参见下面的值，它在启动SMARTCAN时也有同样显示。
        SmartCan Status:          	Enabled	
        Drives Protected:              	C:(SmartCan is Empty)	
        Files Protected:          	All file except those within those extensinos TMP，SMP，INI	
        Archive Files:            	Not Protected	
        Files Deleted After:      	5 days
        9.恢复(Unerase)
        如果无意删除了有用的文件，可以使用恢复命令来将其恢复。因为在删除文件时，只是删除了文件名和释放了文件的链表，这使得文件的恢复成为可能。由于被删除文件释放的空间很可能被其它文件占据而覆盖了原来的数据，因此在文件被删除之后，应尽可能早地恢复它(如果有这种必要)。
        命令格式：
        UNERASE ［pathname］［/IMAGE|/MIRROR|/NOINFO］［/NOTRACK］［/SKIPHIGH］
        UNERASE ［pathname］［/SMRATCAN］［/NOSMARTCAN］［/SKIPHIGH］
        pathname              用于恢复的文件名
        /IMAGE                使用映像恢复信息(不使用镜像信息)
        /MIRROR               使用镜像恢复信息(不使用映像信息)
        /NOINFO               映像和镜像恢复信息都不使用
        /NOTRACK              不删除追踪信息
        /SMRATCAN仅           恢复存放在SMARTCAN中的文件
        /NOSMARTCAN           不恢复存放在SMARTCAN中的文件
        /SKIPHIGH             跳过正在使用的高端内存
        10.磁盘格式化恢复(Unformat)
        磁盘格式化恢复功能可以恢复被格式化了的硬盘、被病毒感染了的硬盘以及由于掉电引起系统故障的硬盘。也可以恢复用安全格式化命令格式化的软盘。
        DOS的格式化命令在格式化硬盘时，实际只重新填写DOS的系统区，包括引导记录、文件分配表和根目录，而并不更改数据区的内容。因此，只要恢复了系统信息区的内容，被格式化了的硬盘就可以复原。
        恢复命令在工作时，会查找IMAGE.DAT(或MIRROR.FIL)文件，如果有这个文件存在，磁盘格式化恢复的工作会变得快而简单。如果没有IMAGE.DAT(或MIRROR.FIL)文件，磁盘格式化恢复的工作也可能进行，只是它的根目录会丢失。
        值得注意的是，如果IMAGE.DAT中的数据太旧，可能恢复的效果还不如没有它的情况。由此可见，更新IMAGE.DAT对日后的许多工作是很有帮助的。
        命令格式：
        UNFORMAT  ［drive:］［/IMAGE|/MIRROR］
        drive:              磁盘格式化恢复的驱动器字符
        /IMAGE              利用IMAGE恢复信息(不使用MIRROR)
        /MIRROR             利用MIRROR恢复信息(不使用IMAGE)
        MIRROR.FIL是DOS 5.0的系统信息存放文件，它与Norton的IMAGE.DAT的格式是相同的。
        (二)、数据安全性组(SECURITY)
        Norton 7.0的数据安全性组由3个独立的实用程序组成。它们用于数据加密，以防止非授权的使用或分析，并且保护它们不被病毒感染等。
        1.磁盘监视器(DISKMON)
        磁盘监视器提供了三种特别功能，用以保护磁盘免受意外或是人为的破坏。
        a)磁盘保护功能，可防止“未授权”的写盘或格式化操作。可用该功能来保护系统区域、用户文件或是整张磁盘，防止它们被任何形式的写操作刷新。这其实也阻止了任意形式的病毒感染，防止因误操作而将数据写到其不应占用的区域中，可阻止用户意外删除或刷新重要文件等。
        b)磁盘灯功能，将当前正被访问的驱动器名显示于屏幕的右上角(图形模式下除外)。有的驱动器的指示灯坏、或不能直视等，磁盘灯功能可以防止用户在系统仍在往驱动器写入数据时，关闭电源或取出软盘。
        c)磁头停放，可以定位硬盘的磁头，以防止在掉电或机器搬动时损伤硬盘。
        在遇到下列情况下之一，可以使用磁盘监视器：
        a)如果用户磁盘上的数据非常重要，一旦被破坏将遭受很大损失。
        b)为防止发生病毒感染。
        c)访问磁盘时，无法看见驱动器的指示灯亮。
        d)硬盘不能自动定位读写磁头。
        命令格式：
        DISKMON /PROTECT+|-［/SKIPHIGH］
        DISKMON /LIGHT+|-［/SKIPHIGH］
        DISKMON /STATUS
        DISKMON /UNINSTALL
        DISKMON /PARK
        /PROTECT+|-              打开或关闭磁盘保护。
        /LIGHT+|-                打开或关闭磁盘灯。
        /SKIPHIGH                不装入高端内存内。
        /STATUS                  显示磁盘保护和磁盘灯的开关特性。
        /UNINSTALL               从内存中清除磁盘监视器。
        /PARK                    对所有驱动器的磁头定位。
        磁盘监视器可以配置成为多种级别的磁盘保护功能(文件、系统区和整个磁盘)。用户必须在全屏幕模式下设置或存储配置文件。当前信息及缺省值保存在配置文件DISKMON.INI中。
        2.磁盘保护(DISKREET)
        磁盘保护命令可以保护机密文件。将这些文件通过加密后存入磁盘，如果没有正确的口令，便无法获取文件的源代码(即解密)。如果没有适当权限，这些文件便不允许被访问。还可以锁住键盘和屏幕，这在用户暂时离开计算机时，防止别人动计算机中的数据。
        磁盘保护命令提供了两种文件加密的方法。
        a)对文件进行单独加密。用户必须首先选中某文件，再利用磁盘保护功能对其完成
        加密，并且提供一条口令。以后每当需要用磁盘保护命令对该文件进行解密时，你必须给出与加密时相同的口令。
        b)划区自动加密。用户可以在某个磁盘驱动器上建立一个实体(entity)，称之为NDisk。一个NDisk看上去象是DOS下的隐含文件，但它的功能却如同一个磁盘驱动器。对它的访问与对磁盘驱动器读写是相同的。但每个存放在NDisk中的文件都是经过自动加密的。NDisk是由口令保护的。每当用户试图打开NDisk时，计算机会自动提示要求用户给出口令。一旦成功打开之后，可随意访问其中的文件。凡进入NDisk的文件会被自动加密，凡从中取出文件则会自动解密。这使得用户可以很方便地访问自己的文件，而不必再输入口令。关闭NDisk，便可以防止没有口令的用户对这些文件的正常访问。
        磁盘保护的许多特性使得对NDisk操作既简单又安全：
        a)可选择在引导期间或被查阅时自动打开NDisk的功能__只要用户能给出正确的口令。
        b)可选择的利用超时或热键自动关闭NDisk。
        c)可选择的对所有试图打开NDisk的审查报告，而无论它们成功与否。
        d)两种加密的选择：快速(quick)和DES(慢速，但是相当安全)。
        磁盘保护命令还使用户可以很方便地锁住你的键盘，清除所有屏幕显示，从而防止无权用户进入你的计算机。
        在CONFIG.SYS中加入命令行
        DEVICE=path\DISKREET.SYS来启动磁盘保护程序。
        在有扩展内存的计算机上，应当在CONFIG.SYS用下面的命令将磁盘保护程序放入高端内存。
        DEVICE=path\HIMEM.SYS
        在首次选择磁盘保护时，必须先建立一个NDisk，它会出现一个对话框，要求输入NDisk名、存入区域、描述信息、NDisk大小、加密方式审查等内容。
        描述信息最多30个字符，用于说明这次建立的NDisk。审查可以在用户打开NDisk磁盘时，告诉你以往打开NDisk的成功和失败的次数，还可以记录他人是否曾试图访问你的NDisk，以及报告最近一次口令的更改情况。
        Diskreet有自己独立的屏幕和下拉菜单，通过菜单可以方便地操作NDisk，如打开、关闭、编辑、更改口令等。
        命令格式：
        DISKREET ［/ON|/OFF|/CLOSE］
        DISKREET /SHOW［:drive］|/HIDE［:drive］
        DISKREET /E:filespec/P:password
        DISKREET /D:filespec/P:password/T:target
        /ON    		允许Diskreet设备驱动程序。
        /OFF			禁止Diskreet设备驱动程序。
        /CLOSE		关闭所有打开的NDisk。
        /SHOW		将隐含NDisk改为显式。
        /HIDE		将显式NDisk改变隐含。
        drive		驱动器字符，缺省时为当前驱动器。
        /E加密(ENCRYPT)〉
        /D解密(DECRYPE)。
        filespec		加、解密的文件名。
        /P			指定一个口令(PASSWORD)。
        password		加、解密用的口令串。
        /T			指定被解密文件
        target		被解密文件的文件名
        NDisk的使用还必须装入Diskreet.sys驱动程序，它必须在系统启动时由CONFIG.SYS装入，在CONFIG.SYS中加入如下命令行：
        DEVICE=DISKREET.SYS［/A20ON］［/NOHMA］［/SKIPHIGH］［/UNLOAD］
        /A20ON  	如果使用高地址存储区，就迫使A20线接通。
        /NOHMA  	不使用高地址存储区(HMA)。
        /SKIPHIGH	跳过正在使用的高端内存。
        /UNLOAD	将DISKREET卸载。
        3.清除信息(Wipe Information)
        清除信息命令能从磁盘中清除数据，它与一般删除文件或格式化不同的地方在于，它要把文件的数据区中的信息也清除掉，使得任何恢复数据的企图都变得不可能，从而达到保护数据的目的。
        清除信息命令可以完成以下工作：
        a)选择清除的区域，清除区域可以是整片磁盘，某个文件或某个区间。
        b)指定写入清除区的数据内容。
        c)指定清除的次数。
        d)选择适合美国政府安全标准的清除算法。
        清除信息中有安全保护措施，可以避免意外的数据丢失。在开始作数据清除之前，会提示用户确认系统警告信息。
        在遇到下述情况之一时，应当使用清除信息命令，而不要使用通常的DOS删除命令或格式化命令。
        a)删除或移动机密文件
        b)对文件的无效区写数据。
        c)格式化有机密数据的硬盘。
        d)将一张可能含有机密数据的磁盘交付他人。
        e)某些应用程序产生的有机密数据的临时文件。
        命令格式：
        WIPEINFO  ［drive:］［/E］［/Gn］［/Rn］［/Vn］［/BATCH］
        WIREINFO  ［pathname］［/N］［/K］［/S］［/GOVn］［/Rn］［/Vn］［
        BATCH］
        drive:  	清除的磁盘驱动器字母。
        /E		仅清除全部被删除和未使用的数据空间。
        pathname	清除的文件名
        /N		非清除方式(删除文件，但不清除它们)。
        /K		清除文件的松驰(slack)空间(文件的内容碎片)。
        /S		清除或删除包括子目录在内的文件。
        /GOVn		按照美国政府标准清除数据(重复模式n次，缺省值是3)。
        /Rn		重复清除n次(缺省值是1)。
        /Vn		指定覆盖清除区域的值(缺省值是0)。
        /BATCH	执行命令时不作任何提示(这适合于用在批处理文件中)
        (三)、系统性能组(SPEED)
        系统性能组通过测试各种硬件特性和改进数据在用户磁盘上的组织方式，提高系统的速度和效率。还以每天为基础，保护你的数据。
        系统性能组由三个独立的实用程序构成。
        1.校准(CALIBRAT)
        校准命令利用执行用于估算和调节的测试程序，来提高用户硬盘上数据存取的速度和可靠性，并且在操作过程中不破坏磁盘上的任何数据。
        为了提高读写数据的可靠性，校准命令测试硬盘上的每个字节，同时把含有错误字节的那些簇作上标记；这样，向磁盘中写数据时，作了标记的坏簇就不被分配来存放数据。先前存贮在这些簇中的数据，都自动地被转移到磁盘中安全可靠的区域里。
        由于校准命令在操作过程中不会破坏磁盘上的任何数据，因而它没有破坏作用，也没有必要预先备份目标盘中的数据。校准命令作低级格式化也是安全的，不会破坏或丢失数据。如果刚好在作低级格式化时出现了电源故障，也可以使用校准命令将磁盘的数据复原。
        校准命令还要对用户的磁盘系统作大量的测试，并且向用户报告测试结果。在遇到如下情况之一时，可使用校准命令：
        a)为了将磁盘的交错率修改成最佳设置。对于给定的磁盘而言，这通常是一次性操作。
        b)用户在从磁盘中读取数据或者向磁盘中写入数据时，总是出现间歇性错误或者有规律的错误。
        c)为了排除越来越严重的硬盘数据存取故障，用户可以每隔一定周期运行一次校准命令。
        命令格式：
        CALIBRAT［drive:］［/BATCH］［/BLANK］［/NOCOPY］［/NOFORMAT］［/NOS
        EEK］［PATTERN:pattern］［/R［A］:pathname］［/x:drives］
        drive:    	需要优化的驱动器字母。
        /BATCH	完毕之后，跳过所有提示，退回DOS状态。
        /BLANK	执行模式测试时封锁屏幕。
        /NOCOPY	不复制当前正在测试的磁道的拷贝。
        /NOFORAMT	只进行模式测试，不执行低级格式化操作。
        /NOSEEK	不执行查找测试。
        /PATTERN	指定用于模式测试的数字。
        pattern	可以选择的模式数为0，5，40或80。
        /R［A］	将结果报告送至(或附加到)某个文件中。
        pathname	报告文件名。
        /x		禁止对指定的驱动器测试。
        drives		不被测试的驱动器列表。
        因为校准命令能够有效地改变用户硬盘的低级格式化(包括硬盘的交错率)，因此第一次使用校准命令时，应在全屏幕模式下运行该程序。在初始校准之后，借助于命令行开关可以周期地对硬盘进行维护性检查。
        2.Norton高速缓存(Norton Cache)
        使用Norton高速缓存命令可以将用户使用最频繁的数据放在内存中。由于这减少了访盘次数和时间，可以改善系统性能，加快应用程序的执行速度。
        因为高速缓存非常有用，而且也不会占用太多的系统空间，尤其是现在的计算机通常都有较大的扩展内存，所以应当在开机时自动启动高速缓存命令。
        自动启动方式可以是下面的方式之一：
        a)在CONFIG.SYS文件中加入命令行
        device=path\nach.exe/install［parameters］
        为了保证兼容性，该语句写在CONFIG.SYS的最后一句为好。
        b)在自动批处理文件AUTOEXEC.BAT中加入命令行
        path\ncach.exe/install［parameters］
        命令格式：
        NCACHE2 ［Install Switches］［［drive:］［Drive Switches］…］
        NCACHE2 ［Reconfiguration Switches］［［drive:］［Drive Switches］…］
        安装开关(Install Switches)：
        /INSTALL      	用缺省值安装高速缓存。
        /OPTIMIZE=S|E|M	根据速度、效率或内存优化高速缓存。	
        /EXT［=n［，m］］	指定用作高速缓存的扩充存储器的大小。单位是K字节。如果省略n，就将所有扩充内存作为高速缓存，m是最小容量。	
        /EXP［=n］		指定用作高速缓存的扩展存储器的大小。	
        /DOS［=n］		指定用作高速缓存的DOS内存的大小。	
        /BLOCK=n		指定高速缓存的块的大小(可以是512，1024，2048，4096或8192字节)。	
        /USEHIGH=ON|OFF	指定是否使用高端DOS内存，使有效内存达到最大。	
        /USEHMA=ON|OFF	指定是否使用XMS HMA，使有效DOS内存达到最大。	
        /READ=n		指定提前读缓冲区最大尺寸。单位是K。	
        /WRITE=n		指定智能化写缓冲区的最大尺寸。单位是K。	
        /DELAY=ss.hh		指定智能化写缓冲区的延迟时间，ss是秒，hh是百分之一秒。	
        /MULTI=ON|OFF	使能或不能多任务智能化写的特性。	
        /QUICK=ON|OFF	ON表示写完后快速返回DOS提示符，OFF表示返回DOS提示符之前，等待所有延迟写操作执行完。	
        /INI=fspec		指定配置路径和文件标识。	
        /REPORT［=ON|OFF］	显示当前高速缓存状态的详细报告。	
        /QUIET		指定报告出错信息是为静态方式	
        /STATUS［=ON|OFF］	显示高速缓存的状态。
        重配置开关
        /RESET   		重新设置高速缓存。不能与驱动器说明符一起使用。
        /UNINSTALL		从内存中拆除高速缓存。
        /SAVE			将当前的高速缓存配置存放在.INI开关所指定的文件中。如果未指定
        /INI			开关，则将其存放在NCAHE.INI文件中，并将其放到装入高速缓存的目录下。
        /DELAY=ss.hh		指定智能化写缓冲区的延迟时间，ss是秒，hh是百分之一秒。
        驱动器开关
        /［-］A    	 	对指定的驱动器，激活/不激活高速缓存。
        /［-］		C对指定的驱动器，使能/不能剩于数据高速缓存。
        /［-］		I对指定的驱动器，使能/不能智能写。
        /［-］		W对指定的驱动器，使能/不能写出高速缓存。

        4.目录排序(Directory Sort)
        目录排序命令调整某个目录下的文件列表顺序，目录排序命令允许用户根据目录名，扩展名，文件建立时间或日期，文件大小进行排序，并且在全屏幕方式下，可以对单个文件或文件组进行排序，或将它们转移至目录列表中所选择的任意位置。
        一个经过排序的目录中，所有子目录都排在前面，后面是目录下的所有文件，但系统文件和隐含文件的位置不变；即这类文件不参加排序。对于启动盘(如C盘)，系统文件或隐含文件改变位置后，可能导致不能启动。
        命令格式：
        DS ［pathname］
        DS sort-keys［pathname］［/S］

        sort-keys
        按指定的关键字顺序进行排序。
        N	文件名	
        E	扩展名	
        D	日期	
        T	时间	
        S	大小	
        /S	子目录排序

        8.文件查找(File Find)
        文件查找命令通过对一个或多个磁盘的所有扇区进行搜索，或对某些文件及子目录进行搜索，来定位丢失的文件、目录和数据。
        文件查找帮你查找误放的数据是非常方便的。如果有某组信息包含在某组文件中，又无法知道具体的文件是哪一个，也可以求助于文件查找命令。
        在文件查找时，可以使用通配符“*”或“?”作为目标文件名。如果未指定驱动器，文件查找命令就只查找当前(即缺省的)驱动器。如果你省略了文件名，那么磁盘上的所有文件名都会显示在一张表中。文件查找命令将根据文件日期，大小，属性等信息来查找文件。
        当某个文件被删除，或者是由于磁盘损坏引起的，使得文件无法用一般的方法恢复，就可以借助于文件查找命令，对找到的数据进行重新剪接和编辑处理。
        命令格式：
        FILEFIND［pathname］［search-text］［/C］［/S］［/CS］［/BATCH］［/A
        ［+|-］］［/HID［+|-］］［/R［+|-］］［/SYS［+|-］］［/CLEAR］［/D［:］
        ［date］］［/T［:］［time］］［/NOW］［/O:pathname］［/TARGET:Drive］
        pathname	搜索的文件参数。注意下列通配符的特别用法：	
        		*.*  搜索整个驱动器	
        		.\*.* 搜索整个目录	
        		*:*.* 搜索所有驱动器	
        search-text		如果搜索文本包含有空格和/或逗号，那它必须整个由引号括起。	
        /C			搜索当前目录	
        /S			包括子目录中的文件	
        /CS			区分大小写(case-sensitive)	
        /BATCH		完成之后自动退出程序	
        /A［+|-］		档案位置位的文件。可设置或清除。	
        /HID［+|-］		隐含文件，可设置或清除	
        /R［+|-］		只读文件，可设置或清除	
        /SYS［+|-］		系统文件，可设置或清除	
        /CLEAR		清除所有属性	
        /D［:］［date］	设置时间(月-日-年)	
        /T［:］［time］	设置日期(小时：分：秒)

        9.文件定位(File Locate)

        文件定位命令与文件查找命令是不同的，前者是通过目录查找文件，后者是字符串内容查找，因此，它可以在未分配的空间中找信息。
        文件定位命令对隐藏文件和系统文件也有效。
        命令格式：
        FL ［drive:］［filename］［/A］［/P］［/W］［/T］［/E:name］［/F［n］］
        drive   		查找文件的驱动器(缺省时用当前驱动器)
        filename		用于查找文件名，可用通配符“*”和“?”。缺省时列出所有文件。
        /A			在所有驱动器上查找文件	
        /P			每屏显示之后，暂停(Pause)	
        /W			按宽(Wide)格式，列出文件	
        /T			只搜索路径(PATH)语句	
        /E:name		只搜索由［name］环境变量指定的路径	
        /F［n］		找出匹配的前面［n］个文件(缺省匹配数为1)
        10.文件大小(File Size)
        文件大小命令用于列出一个文件或一组文件的文件大小，也显示这些文件的和的大小以及文件组中的松散(slack)空间。当期望把这些文件拷贝到某一目标驱动器时，可先查看目标盘的空闲空间的大小。如果目标磁盘空闲空间的大小正好与要拷贝的文件组的大小的总和相等，则往往目标盘装不下这些文件，因为文件分配是以簇为单位(若干个扇区为一簇，簇的大小不都相同)，因此要占用一些额外的空间(称为内部碎片)。
        命令格式：
        FS ［Pathname］［target-drive:］［/P］［/S］［/T］
        Pathname		文件名或目录名
        target-drive		测试目标驱动器是否有足够的空间来存放选定的文件
        /P			每屏显示后暂停
        /S			包括子目录
        /T			只显示总和(概要信息)。
        11.行打印(Line Print)
        行打印命令可以直接从DOS命令行打印文本文件。它的功效相当于使用DOS的内部命令TYPE。
        如：
        TYPE pathname＞prn
        就可以将文本文件从打印机上输出。但行打印命令比TYPE命令有更强、更完善的功能，它能够打出行号、定义页长、附加页头等。文本的规范化输出就十分方便，不需要使用其它编辑程序来帮助了。
        命令格式：
        LP pathname［output］［/Tn］［/Bn］［/Ln］［/Rn］［/Hn］［/Wn］［/TABn］
        ［/SPn］［/PAn］［/n］［/80m132］［/HEADERn］［/WSmEBCDIC］［/PR:xx］［/SET:pathname］［/PS］
        pathname		要打印的文本文件	
        output			打印机(如LPT1)或文件名，缺省值为PRN。	
        /Tn/Bn/Ln/Rn	顶部、底部、左边和右边空格(缺省为3、3、5和5)	
        /Hn/Wn		每页的高和宽(缺省为66和85)	
        /TABn			制表间隔(缺省为8)	
        /SPn			行间距(缺省为1)	
        /PAn			开始页号(缺省1)	
        /N			打印时附加行号	
        /80m132		打印宽度为80或132列(IBM)	
        /HEADERn		HEADER类型(0，1，2)：	
        			0：无页头	
        			1：当前日期和时间占1行(缺省)	
        			2：当前日期和时间以及文件日期和时间占2行	
        /WS			打印WordStar文件	
        EBCDIC	EBCDIC		编码文件
        ［/PR:xx］		打印机类型，xx是：	
        			TT tty	
        			GE generic	
        			EP epson	
        			PR proprinter	
        			QU quietwriter	
        			TO toshiba	
        			LA laserjet	
        			PO postscript	
        /PS			为PostScript打印机产生输出(忽略/80m132开关)	
        /SET:pathname	带有打印机设置串的文件。/SET所指定的文件中的设置串按下列格式使用普通字符和特殊字符：	
        			\nnn ASCII码，nnn为要发送的十进制数。必须是3位数字。不足3位，前面补0。	
        			\X 控制码。X是要发送的控制字母。\A即为Ctrl+A。
        12.改变目录(Change directory)
        改变目录命令可以通过目录树的图形结构来管理目录。这对于多驱动器(包括逻辑驱动器)，多个与多级目录的管理都十分的方便。除了可以改变目录之外，本命令还可以创建、修改(即换名)、删除目录，我们把卷标号也视为目录，则该命令还可修改卷标，还可以打印目录树等。
        命令格式：
        NCD     		［drive:］［pathname］［/R］［/N］
        NCD			［drive:］/V:label
        NCD			［drive:］/L［：ouptput］［/G|/NG］［/T］［/A］［/P］
        NCD			［drive:］MD|SIZE pathname
        NCD			［drive:］/RMTREE［/BATCH］
        NCD COPY		［drive:］|GRAFT source destination
        drive			有要处理的目录的驱动器
        pathname		改变、修改、创建或删除的目录名。
        source			要拷贝的源目录名。
        destination		目标目录名。
        /R			重新读入磁盘目录名。
        /N			不存放目录树信息文件
        /V［：label］	放卷标号到磁盘上。
        /L［：output］	显示/打印目录树(打印到一个文件)。
        /G			按图形方式显示目录树
        /NG			按非图形方式显示目录树
        /T			显示目录树的文件总数和总大小。
        /A			显示所有驱动器(除软盘之外)。
        /P			每屏显示满之后暂停。
        MD|RD			创建或删除子目录
        SIZE显示一棵目录树的大小。
        RMTREE删除一棵目录树，将下属子目录及其中的文件一起删除。
        /BATCH	执行时不带提示(用于批处理文件的删除功能)。
        COPY|GRAFT	拷贝或移动目录树。
        /DELETE决定	拷贝完之后，是否删除源目录及其文件。
        13.安全格式化(Safe Format)
        安全格式化命令可以不用的方式对磁盘进行格式化，包括：
        a)安全格式化
        b)快速格式化
        c)DOS格式化
        用安全格式化格式磁盘，比用DOS的格式化命令更快、更安全。用安全格式化命令格式化磁盘时，它先要分析磁盘，并将包括文件分配表、文件目录在内的重要数据存放到磁盘的最末端的若干个扇区，如果一旦发现格式化命令有误，则可以用这些数据来恢复系统。
        安全格式化命令格式化的磁盘必须是曾经用DOS格式化命令格式化过的。如果是对一张全新的磁盘进行安全格式化，则会看到如下提示：
        这张磁盘先前没有被格式化过，你必须用DOS格式化方式先将其格式化。
        安全格式化在存放了系统信息之后，就从高磁道开始向低磁道格式化。
        在格式化命令工作之前，可以选择驱动器、磁盘类型、格式化方式、是否带系统文件以及是否存放IMAGE信息。
        在它的配置对话框中，还有两个选项：
        a)缺省磁盘符时，是否提问。
        b)是否允许格式化硬盘。
        缺省值要提问和不允许格式化硬盘，这对防止硬盘被格式化是很有帮助的。
        命令格式：
        SFORMAT  	［drive:］［/A］［/S|/B］［/V:label］［/Q|/D］［/1|/4|/8］［/N:n］［/T:n］［/F:size］［/U］
        drive   	目标盘所在的驱动器
        /A		自动方式(在批处理方式是有用的)
        /S		拷贝系统文件到目标盘
        /B		保留存放系统文件的空间
        /V:label	写一个卷标号到目标盘上
        /1		只格式化单面
        /4		在一个1.2M的驱动器上格式化360K的盘	
        /8		每磁盘格式化8个扇区	
        /N:n		每磁道扇区数(8，9，15，18，或36)。	
        /T:n		磁道数(40或80)	
        /F:size	磁盘容量(比如：/F：720将创建720K的盘)	
        /Q		快速格式化	
        /D		DOS格式化	
        /U		无条件格式化(对存在的数据不给警告)
        14.系统信息(System information)
        系统信息命令是个功能强大的实用程序，它有自己的一套菜单系统，包括：
        File  System  Disks  Memory  Benchmarks  Help
        等5个方面的内容，几乎可以获取计算机现行状态的各种信息，CPU速度、硬件配置、存储器使用、中断地址、访盘速度以及CMOS状态等多种参数。
        下面是一个曲型的系统概要信息报告：
        ┌───────┐
        │System Summary│
        └───────┘
        _____Computer_____
        Computer Name:IBM AT or compatible
        Built-in BIOS:AMI,Wednesday,November 11,1992
        Main Procesor:80486DX,67MHz
        Math Co-Processor:(Built-In)
        Video Adapter:VGA,Secondray:None
        Mouse Type:Serial Mouse,Version 6.10
        ______Disks_____
        Hard Disks:208M,335M
        Floppy Disks:1.2M,1.44M
        ______Memory_____
        DOS Memory:640K
        Extended Memory:7,168K
        Expanded Memory:OK
        _____Other Info_____
        Bus Type:ISA(PC/AT)
        Serial Ports:3
        Parallel Ports:1
        Keyboard Type:101-Key
        Operating System:DOS 3.30
        通过对话框向前、向后翻屏，可以看到更多、更详细的内容。每一屏的内容都可以选择打印功能将其打印出来。打印时有对话框，请求输入文件名，文件名缺省值是SIREPORT.TXT。
        命令格式：
        SYSINFO  ［drive:］［/ATUO:n］［/DEMO］［/TRS］［/N］［/DI］［/SUMMARY］［/SOUND］
        drive   	需要从中获取信息的目标驱动器，缺省时为当前驱动器
        /AUTO:n	在所有信息屏幕间循环显示，每屏之间停顿n秒。
        /DEMO		DEMO模式，在基准测试和屏幕汇总之间循环。
        /TRS		仅显示活动的TSR程序表
        /N		不作内存检测
        /DI		驱动器信息(Drive Information)屏幕汇总显示。
        /SUMMARY	系统信息屏幕汇总显示
        /SOUND	在CPU测试期间，有声音。
        15.文本搜索(Text Search)
        文本搜索命令可以搜索单个或多个文件，也可以搜索不包括在文件中的文本。如我们对某些文件中的内容感兴盛，而不能辨认是哪个文件，则用文本搜索命令是比较方便的。比如，在FOXBASE的PRG文件中，有某个变量名因某种原因需要一起改变，就可以用文本搜索命令来查找它们。如果一个文件被删除，它的文件名已被覆盖，但假定其内容未被覆盖或未被全覆盖，则可以用文本搜索来恢复它的全部或一部分。
        命令格式：
        TS ［pathname］［search text］［/S］［/T］［/A］［/CS］［/EBCDIC］［/WS］［/LOG］
        TS ［search text］/D/E［/Cn］［/A］［/CS］［/EBCDIC］［/WS］［/LOG］［F:filename］
        pathname    	要搜索的文件名
        search 	text用于搜索的文本串
        /S		搜索包括子目录
        /T		设置非对话形式的信息汇总模式
        /Cn		从第n个族开始搜索
        /D		搜索整个磁盘
        /E		仅搜索磁盘中被删除文件的空间
        /A
        /CS		要区分大小写
        /EBCDIC	搜索是EBCDIC编码的文件
        /LOG		格式化输出，以便于打印或存放文件
        /WSWordStar，不使用扩展字符
        /F:filename	输出文件到保存磁盘族?
