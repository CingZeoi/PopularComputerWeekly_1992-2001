找回“失落”的世界──硬盘数据恢复大法

鸣涧 
2001年 35期

硬盘中的数据资料被误删除！硬盘分区信息丢失！硬盘被误格式化了！CIH病毒光临了我的家门！硬盘数据的丢失或者被破坏，是如此让众多朋友为之头疼。很多初学电脑的朋友面对已经丢失的数据文件，往往手足无措，不知如何是好，这时掌握一些数据丢失恢复技术，就显得非常重要。
　　#1为什么会丢失数据？
　　首先我们还是从硬件介质方面作一些简单的介绍。如果系统无法自检到硬盘，可能是主板已经损坏，那么你不用担心，这对硬盘数据丝毫不会发生影响。但是如果故障出现在硬盘的控制电路还可以修复，如果是出现在磁头或者盘片，数据恢复的可能性就不是很高了。
　　排除硬件故障，就要考虑软件故障原因了，造成硬盘资料丢失的大多数原因都来源于误操作，但是需要视其操作行为，恢复的可能性也是不同的。
　　#1    FAT32/16分区误删除文件的恢复
　　    文件删除是最简单和最常见的数据损坏方式，它的基本原理是把文件的首字节改为E5H，这时并没破坏文件本身，因此是可以恢复的。但是你如果在文件原存储位置重新建立新文件后，一般来说，就无法恢复了。
　　如果需要将误删除的文件进行恢复，可以使用Recover4all，它是一个可以在Winows9x下恢复误删除文件的工具。
　　软件基本信息：
　　最新版本：1.04
　　文件大小：252 KB
　　软件授权：免费软件
　　使用平台：Windows 9X/Me
　　下载地址：http://www.recover4all.com
　　①解压后直接运行rec4all.exe即可打开其主界面窗口（^35020201a^），在主菜单下方的盘符工具按钮中选择相应的盘符，这时它即可自动扫描当前系统下已经被删除的文件，完成以后即可显示出现在右侧的窗口中，同时会在“Recovey Chances”列表项中标记出文件可恢复情况。如果标记为“good”，说明文件是可以被恢复的。
　　②选中要恢复的文件，并点击工具栏的“Recover”按钮，此时会出现一个对话框（^35020201b^），选择恢复文件需要存放的路径后，单击确定按钮即可恢复被删除文件了。
　　#1    NTFS分区误删除文件的恢复
　　很多朋友使用的是Windows2000或者Windows NT系统，这两种操作系统可以使用NTFS分区，在此分区下被误删除的文件。需要说明的是，File Scavenger只能恢复NTFS分区上被删除的文件，同时可以恢复格式化后的分区文件，但是无法恢复被格式化后的分区。此外，使用File Scavenger需要用户在Windows2000/NT系统中以系统管理员身份登录。（注：其中不包括在Windows 2000 Server系统下的文件恢复。）
　　软件基本信息：
　　最新版本：1.40
　　文件大小：1228KB
　　软件授权：共享软件
　　使用平台：Windows NT
　　下载地址：http://www.onlinedown.net/FILESCAVENGER.HTM
　　安装并启动File Scavenger以后，即可进入其主界面窗口（^35020201c^）。
　　①在搜索条件（Serach for）中，可以设定多种文件搜索格式，比如.exe、.doc等。
　　②设定完成搜索格式后，并在“On Drive”中设定好搜索的盘符。
　　③然后在“Output Director”中设定好恢复后文件存放的路径。
　　④最后即可点击“Search”按钮开始按照条件进行查找。
　　⑤如果此时选中了“Exhaustive Sear”选项，可以指定搜寻分区的簇大小以及搜索簇的范围。
　　搜索完成后会在下面的搜索窗口中显示搜索的结果，只要选中要恢复的文件后，并点击“Recover”命令按钮即可将文件恢复到指定路径中了。
　　#1    误格式化与误删除分区表的恢复
　　    有的朋友曾经碰到过不慎将硬盘原分区信息误删除了，或者将硬盘误格式化，造成分区表破坏无法读取数据资料。如果出现这种情况，建议首先关机，不要轻易对硬盘数据资料进行写操错，否则会增加数据恢复的难度。遇到这种情况可以使用EasyRecovery来进行恢复。（注：其中不包括在Windows 2000 Server系统下恢复。）
　　软件基本信息：
　　最新版本：5.11
　　文件大小：2884KB
　　软件授权：试用软件   
　　使用平台：Win95/98/NT/2000
　　下载地址：http://www.onlinedown.net/EASYRECOVERY.HTM
　　EasyRecovery可以修复主引导扇区（MBR）、修复BIOS参数块（BPB）、修复分区表、修复文件分配表（FAT）或主文件表（MFT），还可以修复根目录。另外，如果你的硬盘因病毒破坏，或者误格式化都是可以使用EasyRecovery进行恢复的。
　　启动运行EasyRecovery进入其主界面窗口，会出现欢迎信息，此时只要单击“Next”按钮即可开始对当前系统进行扫描。
　　扫描完成以后，会出现一个系统概览对话框（^35020201d^），在主窗口中会显示当前系统中硬盘的分区情况，如果发现其中包括“Unknown File System Type”类型的分区，那么他们就是被Fdisk之类分区工具误删除的FAT分区。此时需要选中被误删除的分区，然后点击“Next”按钮进入下一步。
　　在新打开的对话窗口中（^35020201e^），会显示被选分区在整个硬盘中的空间分配情况，另外我们还可以手工设定分区的开始（Start Sector）和结束（End Sector）扇区的数值，一般情况下可以无须更改，即可点击“Next”按钮进入下一步。
　　在接下来的步骤中，可以选择文件系统类型和分区格式，其中文件系统类型包括FAT12、FAT16、FAT32、NTFS和RAW。需要说明的是，RAW格式可以用于修复无文件系统结构的分区，可以对整个分区的扇区进行扫描，并找回保存在簇中的小文件或连续存放的大文件等。另外，在分区扫描模式中提供了“Typical Scan”和“Advanced Scan”两种模式，其中Typical模式可以指定分区结构信息，Advanced 模式可以彻底扫描全部分区的结构信息。完成设置以后，可以点击“next”按钮即可进入分区信息的恢复状态（^35020201f^）。
　　在完成分区信息的恢复后，EasyRecovery会将修复后的文件按后缀名进行分类并列表显示，此时我们可以对文件进行标记，只要在文件或者文件夹前面选取并打钩（^35020201g^），并在Destination后面的文本框中输入恢复后文件保存的位置为“c:\mydoc”，完成以后单击“Next”按钮，此时系统将弹出一个提示对话框，点击“Yes”按钮即可选取目录保存，最后，当标记文件被复制完成以后，就可以到c:\mydoc文件中找到被恢复的文件了。
　　#1    CIH病毒破坏后的恢复
　　    CIH病毒破坏了硬盘，使硬盘主引导记录和分区表损坏，硬盘不能引导或软盘引导也不能进入硬盘，造成硬盘数据丢失而无法正常读取。如果我们以前使用过KV3000/B命令在软盘上备份过主引导记录，这时可用 KV3000/HDPT.DAT命令恢复硬盘主引导记录。如果没有备份过硬盘主引导信息，可以使用KV3000快速重建硬盘分区表。
　　①首先可以使用软盘引导系统后，并执行KV3000，出现主菜单后，可以按下F10键，这时就可以对系统的有关参数和硬盘分区表进行快速测试，这时系统会出现如下提示信息：
　　Hard Disk Partition Table - Error
　　Fix Hard Disk Partition Table or Sector（Y/N）﹖
　　这里的意思是表示“硬盘分区表错误，是否修复硬盘分区表或C盘引导区”。
　　②可以按下“Y”键进行修复，此时会开始自动检测，最后会出现提示将坏分区表保存到软盘上的提示信息：
　　Insert a formatted diskette into drive A， Pressed“Y”to save “error” Partition table floppy filename HDPT.VIR，“N” to exit continue（Y/N）。
　　③插入软盘后，并按下Y键，系统即可开始修复。成功修复后，可以按任意键返回DOS状态提示符，然后重新启动计算机。
　　④再次使用KV3000软盘启动系统后，会发现被CIH病毒破坏的数据已经正常恢复了，然后可以使用KV3000将硬盘中的病毒清除，并使用“SYS c:”，就可以完成受损数据的恢复了。
