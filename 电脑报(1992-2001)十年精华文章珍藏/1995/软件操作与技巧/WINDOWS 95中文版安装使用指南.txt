        一、概述
        WINDOWS  95中文版是MICROSOFT  WINDOWS的中文升级版本，它在西文WINDOWS  95基础上进行了内核双字节化，因此，其速度、功能与西文WINDOWS  95无异。另外，它同WINDOWS  3.X不同，它不仅是一个图形用户界面(GUI)，而且是一个完全的32位中文操作系统， 适应了当今的硬件潮流。因此486DX2/66将成为1995年运行WINDOWS  95的标准处理器。
        WINDOWS  95中文版有以下特点：
        1.支持即插即用(PLUG  &  PLAY)特性，插入一个即插即用新的硬件，则WINDOWS  95在启动时均能感知，并自动搜索SYSTEM目录中有关驱动程序，减少了人工配置步骤，给用户带来了极大的方便。
        2.新型WINDOWS  95的用户新视图，考虑了WINDOWS  3.1向WINDOWS  95平稳过渡的特性。
        (1)  WINDOWS  95界面中充满了WINDOWS  3.1用户可以立即识别出的符号和控件，例如，WINDOWS  95的下拉菜单仍在一个菜单选择指示它打开一个子菜单之后出一个向右的三角，并用省略号(...)表明一个菜单选择将打开一个对话框。WINDOWS  3.1用户很容易识别WINDOWS  95对话的复选框、单选框和命令按钮。WINDOWS  3.1的大多数键盘命令，例如ALT＋F4、CTRL＋ESC等在WINDOWS  95中的作用相同。
        (2)  WINDOWS  95在安装完毕后将在屏幕上出现如下图标：
        ┏━━━━━━━━━━━━━━━┓
        ┃  我的电脑                    ┃
        ┃回收站                      ┃
        ┃收件箱                      ┃
        ┃安装The  MicroOsft  Network ┃
        ┗━━━━━━━━━━━━━━━┛
        在WINDOWS  95桌面上的图标履行许多不同的任务，桌面上一个图标可以代表一个单一的对象，如一个应用程序，就象WINDOWS  3.1中的应用程序图标，或一个文件。它还代表某种容器，例如“我的电脑”、一个文件夹或一个磁盘驱动器等。一个文件夹可能包含文件、应用程序或其它文件夹。实际上，文件夹确实与DOS和WINDOWS  3.1中的子目录具有相同的意义。
        我们平时用得最多可能是“我的电脑”和“回收站”。“我的电脑”实际上把用户电脑上所有驱动器(如A、B、C、D等)包容在此图标中。另外，“我的电脑”中还包括了“控制面板”、“打印机”等。WINDOWS  95的屏幕或打印机的配置图标。每个驱动器图标相当于软盘或硬盘分区中的内容，例如D：图标则包容D盘所有目录和文件，因此，要执行某个文件，则只要打开该盘图标下的该文件所在目录即可。例如，要打开原中文WINDOWS  3.1的主组窗口，则只要打开D： \PWIN目录，然后，执行该目录下的CMAINGRP图标，则WINDOWS  95将执行开始菜单的快捷转换方式， 显示出WINDOWS 3.1的主组窗口，十分方便。
        在WINDOWS  95桌面的最下一行(在WINDOWS  95中被称为“任务条”)开始处有一个“开始”图标，在它上面点一下，将出现一个上托的菜单，它包括了“程序”(附件、启动、WINDOWS的资源管理器、MS  DOS方式)、“文档”(15个文件菜单)、“设置”(硬件和软件配置)、“查找”(相当于UNIX的GREP)、“帮助系统”、“运行”(相当于WINDOWS  3.1的FILE-RUN菜单)和“关闭系统”。每当您在WINDOWS  95中运行一个程序或打开一个任务，均在此任务条中依次显示打开的任务，要找到一个打开的、被缩小的窗口的唯一方法就是寻找任务条上的指令按钮。3.WINDOWS  95允许用户命名255个字符的文件名，例如“SECOND  QUARTER  SALES  REPORT”都是合法的文件名。
        4.WINDOWS  95提供了一个集成文本编辑器和字处理器的“写字板”(WORDPAD)，它可处理文本文件、格式化文档(WORD  6.0  FOR  WINDOWS和WINDOWS  WRITE)。
        5.在WINDOWS  95中文版中运行汉化DOS应用程序时，不再需要DOS上的汉字操作系统，且在运行时可使用WINDOWS  95中文版的汉字输入方法。
        6.WINDOWS  95中文版提供了全拼输入法，双拼输入法，郑码输入法，内码输入法和智能ABC输入法，并可利用其提供的通用编码编译器CONVMB自己扩充其它汉字输入法，如五笔字型输入法等。
        “开始”菜单的最右边有一个“En”图标，这是WINDOWS  95默认的输入法状态-英文输入法(English) 。WINDOWS  95的汉字输入法可在此激活，即点一下此图标，则在出现一个上托菜单，显示WINDOWS  95中提供的输入法，您可从选择一个，例如“拼音输入版本2.0”。在选择该输入法后，“En”图标将变为您所选择输入的图标，但看不到输入法行。若在“开始”按钮的右边上边缘处点一下， 则将出现一个输入法提示窗口，有全角/半角、输入法名称、键盘图标等。此时，若您在“附件”中打开“写字板”，就可在“写字板”中输入汉字了。与WINDOWS  3.1中文版不同的是，汉字输入框与光标键有很大的关系，即汉字输入框将紧随光标键移动，在不输入汉字时，汉字输入框将不出现。另外，在输入法图标左边有一个图标，它的作用是否显示IME输入法，它有“显示IME输入状态”和“关闭IME输入状态”。为了清楚起见，我们一般应选择“显示IME输入状态”，以便知道现处于哪一种输入法状态。
        7.从WINDOWS  95开始告别了多媒体驱动程序MSCDEX.EXE，它提供一个独立的32位保护模式的CD-ROM文件系统CDFS，此文件系统比MSCDEX快得多，并有自己的内存高速缓存，类似于32位硬盘驱动器(VFAT)的高速缓存。
        8.WINDOWS  95用REGISTRY代替了WIN.INI、SYSTEM.INI和REG.DAT，它是一个比WINDOWS  3.1的REG.DAT更为可靠的数据库系统，不易遭到外部损坏。WINDOWS  95的REGISTRY主要由三个文件组成的，即SYSTEM.DAT、USER.DAT和POLLICES.DAT，它们为只读、隐藏和系统文件属性的二进制文件，一般不能被DOS命令删除，也不能被一般编辑软件进行编辑修改。
        9.在WINDOWS  95中增加或增强了一些DOS命令：
        (1)  所有的DOS文件命令，如DIR、XCOPY、REN等被扩充使用长文件名，要使用长文件名只要用双引号“”括起来就行：
        COPY  “NEW  DOCUMENT”  “ANOTHER  DOCUMENT”
        (2)  DIR命令已经扩充配合了长文件名，若用DIR命令显示文件列表时，我们可发现列表的前六列显示了通常的DIR信息，新添的一列显示了文件的长文件名。 另外，DIR命令也多了一个新的命令行开关：/V，此开关将产生一个长的目录列表，其中包括文件属性、最后访问的时间和文件最后修改的信息等，
        (3)WINDOWS  95提供了一个新命令START，此命令支持WINDOWS  95文档趋势及消除DOS和WINDOWS应用程序的差别。START命令可以从DOS行命令启动WINDOWS和DOS应用程序 (DOSVM提供的命令提示符)，其方法是指明要启动的程序名或文档名。
        START  WORD
        START  “LETTER  TO  DAVE.DOC”
        以上每条命令都打开WORD  FOR  WINDOWS，但第一条只是启动WORD-MS  WORD的装入，并显示空的文档。第二条命令启动WORD，并让它装入或打开LETTER  TO  DAVE，其扩展名为DOC是告诉WINDOWS  95，它是WORD的文档。
        若用户使用了DOS  START命令启动DOS应用程序，WINDOWS95将创建一个新的DOS  VM(DOS虚拟内存)，并在其中运行DOS应用程序。
        10.WINDOWS  95中文版可识别PDOS  6.22中文版，并可自动汉化WINDOWS  95中的DOS  VM窗口。

        三、WINDOWS  95中文版的安装
        如果还没有安装WINDOWS 95，则有关安装的一些附加信息可在Setup. txt文件中找到， 此文件在WINDOWS 95的第一张安装盘上，可以使用WINDOWS 3.1的记事本进行阅读。若您购买的是WINDOWS 95光盘版本，则可在\Win95目录下找到此文件。
        (一)安装需求
        安装WINDOWS 95的需求如下：
        1. 最小内存容量: 运行WINDOWS 95中文版最小内存容量应达到6MB。最好为486DX2/33以上配置，8MB以上内存。
        2. 常规内存: Setup安装程序需要至少432,128字节的常规内存才能运行。
        (二)WINDOWS  95中文版的安装
        1.首先决定如何安装WINDOWS  95中文版
        WINDOWS  95中文版在安装过程中，将提供了两种安装方法，即
        (1) 勇敢地踏入WINDOWS  95这个新的领域，从现有的WINDOWS系统中升级到WINDOWS  95中文版系统上。
        实际上，该选择是明智的选择，因为在这种升级过程中，WINDOWS  95中文版的安装程序SETUP将把用户的程序管理器中的全部现存的程序组自动地转换为WINDOWS  95的格式，在这里的“全部”的意思就是每一个程序组，包括STARTUP程序组。在安装完WINDOWS  95后，所有WINDOWS  3.1的应用程序均可WINDOWS  95中直接运行，而无需要重新安装和配置这些程序。以升级的方法安装WINDOWS  95，将保留和转换配置，例如用户原来使用的那种WINDOWS屏幕锁定程序和它的设备。若用户是在网络中升级WINDOWS  95中文版，则它还会转换一些网络信息，例如计算机和工作组的名字等。
        (2) 仅在必要的时候，才选择第二种安装方法即“双启动安装”。
        该安装方法主要目的是为了满足那些想要把WINDOWS  3.X中文版移植到WINDOWS  95中文版，但要立刻接受新的系统而又有怀疑的用户。当使用WINDOWS  95的建立双启动安装时， 实际上安装的WINDOWS95同现在的DOS和WINDOWS系统并存于一个系统之中，双启动安装选择项使用户原有的WINDOWS  3.1系统保持完整且不受任何干扰。结果是，可以忽略WINDOWS  95的启动过程，且使用原来的DOS和WINDOWS  3.1系统，就好象从未安装过WINDOWS  95一样。
        由于双启动安装方式不会把现有的WINDOWS  3.1程序组、打印机或其它设备驱动程序转换为WINDOWS  95格式，因此，要在WINDOWS95上运行原来WINDOWS  3.1的程序， 必须重新安装WINDOWS3.1的应用程序，否则，将无法运行。于是，在计算机中将会出现两个同样程序的拷贝，占用了大量磁盘空间。
        2.安装前的一些准备工作
        (1)  在开始安装之前应该关闭已运行的屏幕保护程序和任何一种防病毒软件，包括新版CMOS中的防病毒软件。
        (2)WINDOWS 95 安装程序将自动运行磁盘扫描程序检查硬盘。 如果遇到问题，安装将会停止，直到解决问题以后，才能继续安装。
        注意: 安装程序加载的磁盘扫描程序如果是 MS-DOS 版本的，可能会检测到长文件名错误，但不能修复它们。安装程序会忽略这些问题继续运行。但一旦安装完毕，应该在 WINDOWS 95 下运行新的 WINDOWS 版 SeanDisk 修复这些错误。
        如果在安装过程中提示运行磁盘扫描程序修复硬盘错误，可以按后面所述的步骤进行修复。 如果在 MS-DOS 或 WINDOWS 的早期版本 (如 WINDOWS 3.1) 下安装WINDOWS 95:，请按下列步骤进行：
        ①退出 WINDOWS。
        ②如果从软盘安装，请将 1 号安装盘插入软驱，然后在命令行中键入:
        a:scandisk.exe /all
        其中 “a:” 是包含 WINDOWS 盘的驱动器。
        如果从 CD-ROM 上安装，请插入 CD-ROM 并键入以下命令 :
        f:\win95\scandisk.exe /all
        其中 “f：”是CD-ROM 驱动器。
        ③按屏幕上的指导，修复磁盘扫描程序发现的所有问题。
        ④启动 WINDOWS，再次运行安装程序。
        如果在安装过程中遇到“内存不够，无法运行磁盘扫描程序”的提示，可以释放一些常规或高端内存， 然后再次运行安装程序。 如果有 MS-DOS 6. X，可以使用MEMMAKER 释放内存。
        如果仍旧出现内存不够问题，或在安装时运行磁盘扫描程序存在其它问题，可在运行安装程序时加上 /IS 选项跳过磁盘扫描程序，即键入下列命令:
        setup /is
        注意: 建议最好不要在安装时跳过磁盘扫描程序。如果跳过，可能会因为硬盘问题而无法安装 WINDOWS 95， 或无法正确运行 WINDOWS 95。如果带 /IS 选项运行 Setup 仍出现问题，可以再试用 /IQ 选项。即键入下列命令:
        setup  /is /iq
        (3) 使用了压缩驱动器的处理
        如果您使用压缩软件压缩过硬盘，那么可能会在安装过程中看见一条信息：“压缩驱动器的主分区没有足够的空间” 。安装程序可能会将部分文件复制到启动盘、启动盘的主分区或WINDOWS 盘的主分区。如果您收到这则消息，您应该释放指定盘上的部分空间，然后再次运行安装程序。可以按下列方式释放空间:
        - 如果在压缩驱动器上安装 WINDOWS，您可以试试将其安装到非压缩驱动器上。
        - 删除主分区上所有无用文件。
        - 如果正在运行 WINDOWS 3.1 而且有一个永久交换文件， 可以试一试减小文件尺寸。即在“控制面板”中双击“386 增强模式”图标，然后单击“虚拟内存”，更改交换文件的大小。
        -  使用磁盘压缩软件为压缩驱动器释放主驱动器上的部分空间。
        ①  DRVSPACE或DBLSPACE压缩驱动器的处理
        如果使用DriveSpace 或 DoubleSpace 压缩过驱动器，请按下列步骤操作:
        1. 退出 WINDOWS。
        2. 运行 DRVSPACE.EXE 或 DBLSAPCE.EXE。
        3. 选取要释放主驱动器空间的压缩驱动器。
        4. 单击“驱动器”菜单中的“更改大小”。
        如果使用的是其它压缩软件，如 Stacker，请参阅该软件的文档。
        注意: 如果安装程序报告的可用空间数和您认为的主驱动器上应该有的容量存在差异， 可能是因为  WINDOWS  为创建交换文件占用了一些空间。  在没有运行WINDOWS时不会占用任何空间。
        注意: 如果在安装过程中创建了启动盘，请确认您的启动盘不是压缩盘。
        ②使用 SuperStor 压缩驱动器的处理

        i. 启动 WINDOWS，然后再次运行安装程序。

        原名				新名
        IO.SYS				IO.DOS
        MSDOS.SYS			MSDOS.DOS

        原名			新名
        COMMADN.COM		COMMAND.W95
        CONFIG.SYS		CONFIG.W95
        AUTOEXEC.BAT		AUTOEXEC.W95

        (3)在 OS/2上安装 WINDOWS 95
        安装程序无法在 OS/2 下运行。 您需要先用软盘启动 MS-DOS，然后从命令行运行WINDOWS  95的安装程序。

        如果在安装完 WINDOWS 95 后想删除 OS/2，请执行下列操作 :
        1. 将有用的数据文件备份到软盘或网络驱动器。
        2. 删除每个 OS/2 目录及其子目录下的所有文件，然后再删除所有 OS/2 目录。
        3. 需要删除下列在根目录下隐藏的系统文件 :
        EA DATA.SF、OS2LDR.MSG、OS2KRNL、OS2BOOT、WP DATA.SF
        请确认“我的电脑” 或“ WINDOWS 资源管理器”中隐藏文件是否可见。您可以这样做: 先单击“查看”菜单，单击“选项”，再单击“显示所有文件”，然后删除上面列出的 OS/2 文件。
        注意: 如果 OS/2 不是 2.0 版的， 例如3.0  WARP版本，则硬盘上的OS/2 文件可能与上面所列的文件不同。另外，根据不同版本的 OS/2，您可能会在根目录下看见下列需要删除的文件:
        OS2DUMP、OS2LDR、OS2LOGO、OS2VER。
        4. 请记住清空回收站以便永久删除这些文件。
        如果安装了引导管理模块，而现在想删除它，可以重新启动计算机，执行下列操作(建议在重启动计算机前打印本文件)。
        1. 出现“引导管理模块”菜单时，选择 MS-DOS，然后运行 FDisk。
        2. 将 MS-DOS 分区 (C) 设置为活动分区。
        3. 退出 FDisk，重新启动计算机。
        如果在安装完 WINDOWS 95 后想再次使用引导管理模块，可以执行下列操作 :
        1. 在“开始”菜单中单击“运行”，然后键入 FDISK。
        2. 选第 2 项-“设置活动分区”。
        3. 键入引导管理模块的分区号。该分区是非 MS-DOS 分区，容量为 1 MB，通常位于分区的内存区的顶端或底端。
        4. 退出 FDisk，按指导重新启动计算机。
        现在您可以随时启动 OS/2，并通过 OS/2 的 FDisk 程序更改引导管理模块的标识名。

        (三)  安装WINDOWS  95中文版的出现问题
        1.安装时的错误信息解释
        下面列出安装过程中可能会遇到的特别信息，并提供解决问题的办法。
        ①SU-0010、SU-0012、SU-0015、SU-0016
        如果在安装过程中遇到这些信息之一， 请参阅前面的关于 OS/2 和WindowsNT 部分，那里有解决该问题的方法。
        ②SU-0011
        如果您的硬盘受口令保护，安装程序将无法正确完成。您必须首先删除口令保护。
        ③SU-0013
        要安装 Windows 95， 您的启动盘必须是 MS-DOS 引导盘。如果启动盘为HPFS或 Windows NT 文件系统，则必须在运行安装程序前创建 MS-DOS 引导区。关于创建MS-DOS 引导区的详细内容，请参阅有关MS  DOS方面的书籍。
        如果带 LANtastic 联网或 SuperStor 压缩，请参阅前面部分介绍。
        ⑤SU0167
        如果在当前 Windows 目录中有一个叫“桌面” 的目录，那么安装程序将无法把 Winodows 95 安装到当前的 Windows 目录中去。 因为 Windows 95  将创建“桌面”目录，用于存放系统文件。如果在安装过程中看见这则信息，应该将“桌面”这个目录换名或将其移走，然后再次运行安装程序。
        2.标准模式信息
        如果您在安装过程中看见下列任何一则错误信息，必须删除 Config.sys 文件中所有的内存管理程序(如 EMM386. exe, QEMM 或 386Max)，然后再次运行安装程序。
        标准模式 : 返回非法的 DPMI。
        标准模式 : MS-DOS 扩展程序错。
        标准模式 : MS-DOS 扩展程序严重错误。
        标准模式 : 错误处理程序的未知堆栈。
        标准模式 : 堆栈溢出。
        注意: 如果仍有问题，请试将 EMM386.EXE 重新加入到 Config.sys 文件中，并且排除所有的地址范围。例如 :
        device=c:\windows\emm386.exe x=A000-FFFF
        如果在安装过程中遇见这些信息或出现莫名的死机现象，可能需要在SmartDrive中打开双缓冲选项。因为部分 SCSI 硬盘和 ESDI 硬盘需要双缓冲。在Config.sys文件的开头加上下列语句就能打开双缓冲功能:
        device=c:\windows\smartdrv.exe /double_buffer+
        其中 "c:\windows" 是到 Windows 目录的路径。
        3.无法读取安装文件 (.CAB)
        如果有多媒体 Cloaking 并且从软盘安装 Windows 95，安装程序可能无法正常运行。如果在安装过程中看见无法读取 .CAB 文件的信息，请按下列步骤执行:
        ①将 Config.sys 和 Autoexec.bat 文件中有关 CACHECLK.EXE 的行删掉。
        ②重新启动计算机。
        ③再次运行安装程序。
        4.安装程序无法备份系统文件
        如果在安装程序保存系统文件时看到这条错误信息，可能是压缩盘上的部分磁盘空间不够。应该删除盘上的无用文件以释放该盘(默认情况为 C)上的部分空间。
        5.此系统配置下无法继续
        如果遇到这样一则消息-“此系统配置下无法继续”，可能是由于磁盘分区太老或不兼容。请在运行安装程序前，备份数据然后重新分区。
        (四)卸载WINDOWS 95中文版
        在安装过程中有一选项， 可以选择保存系统文件，以便日后可以卸载Windows95。若您想以后能卸载Windows 95，请选择“是”。安装程序会把您的系统文件保存在一个压缩的隐含文件里。若不想以后能卸载Windows 95，请选择“否”。
        如果是在下列情况下进行的安装，您将看不到安装选项：
        - 在早期的Windows 95版本上升级安装。
        - 安装到一个新的目录。
        - 在低于MS-DOS 5.0的版本上运行。
        注意: 卸载文件必须保存在本地硬盘驱动器上，不能把这些文件存放在网络驱动器或软盘上。如果您有多个本地驱动器，可以任选一个来存放这些卸载信息。
        要卸载Windows 95并使您的计算机完全恢复到以前的MS-DOS和Windows 3. x版本，可以使用如下操作：
        1. 单击“开始”按钮，指向“设置”，然后单击“控制面板”。
        2. 双击“添加/删除程序”图标。
        3. 在＂安装/卸载＂标签中，选择Windows 95，然后单击“删除”。
        或者，当您在启动Windows 95时遇到了问题，可使用您的启动盘来启动您的计算机，然后在启动盘上运行UNINSTAL命令。
        注意： 卸载程序需要关闭Windows 95系统。如果您的计算机在关闭Windows95系统时出现了问题，请重新启动机器，在出现“正在启动Windows 95”信息时，按F8键，然后选择Command Prompt Only，在命令提示符下运行UNINSTAL命令。
        如果没有将文件保存到 C 盘，可以使用 /W 选项指定文件所在的驱动器。例如:
        uninstal /w e:
        其中 e: 是保存旧系统文件的驱动器。
        如果在运行Windows 95时您想移走这些卸载文件，以释放6MB 到9MB 磁盘空间，可按以下步骤执行：
        1. 单击“开始”按钮，指向“设置”，然后单击“控制面板”。
        2. 双击“添加/删除程序”图标。
        3. 在＂安装/卸载＂标签中，选择旧的Windows 3.x/MS-DOS文件，然后单击“删除”。
        以后您将不能卸载Windows 95。

        1.WINDOWS  95的DOS窗口的汉化
        下面就是WINDOWS  95中文版中的DOS窗口的汉化方法：
        (1)如果已安装MS-DOS 6.22中文版，Window 95的DOS窗口将升级为中文方式，并自动更新原DOS系统的一些程序模块，  更新的模块放在Windows95安装路径下的COMMAND目录中。
        (2)请确保MS-DOS中文环境的安装路径正确设置在autoexec.bat文件中。
        (3)请不要在原MS-DOS中文环境的安装路径下启动PDOS.BAT。
        (4)请注意不要把中文DOS环境加载到全局。         (6)当从窗口方式切换到全屏方式时， 中文方式状态行并不刷新，用户按下任意一键后，状态行刷新。  (7)在窗口式DOS中文环境下，不支持 CTRL+F10 功能。
        (8)在Windows 95的DOS中文方式下，不支持网络打印。
        (9)在DOS屏幕尺寸被设为34/50行显示方式下加载中文DOS模块, 屏幕将会自动切换为VGA的标准25行方式。
        (10)当在DOS自带的编辑器下激活CTRL+F10时，有时会挂起系统，若用户持续按键，系统会恢复正常。
        2.显示驱动程序
        (1). Windows 95会自动安装自己的显示驱动程序，不保留任何Windows 3.1的显示驱动程序。Windows  95的显示驱动程序支持的分辨率和颜色可能与Windows3.1支持的不一样。  Windows  95支持Windows  3.1的显示驱动程序，  但不支持Windows 3.1的VDDs。请不要在Windows 95上安装Windows 3.1的显示驱动程序。
        (2) . 如果您的显示卡上有S3显示适配器，在代码页437下，当在DOS的窗口方式和全屏方式之间切换时， 显示器可能会出现黑屏幕。这可能是因为在Window 95下，S3显示驱动程序设置的水平扫描频率过低，而您的显示器不支持这种扫描频率。为解决这个问题，可以从\drivers\display\s3vxd目录下拷一个新的S3显示驱动程序(S3.VXD)到您的\Windows\system目录下。
        3.声音驱动程序
        IBM ThinkPad 230Cs的声音驱动程序在Windows 95安装和启动时会引起错误。Windows 95一旦发现该驱动程序的日期早于5/22/95，则将它从system.ini 文件中删去。
        4.一般注意事项
        下面包括特定系统配置的注意事项， 这些配置可能会导致 Windows95 的安装失败。
        (1) Sager NP8200 或 Wedge Technologies 466/DX2
        如果在 Sager NP 8200  或  Wedge  Technologies  466/DX2  笔记机上安装Windows 95， 您将无法再次使用计算机，即使您重新安装以前的 Windows 版本也不行。
        (2) Micron P90/1000 和 Zeos P90/100
        如果在这些计算机上遇到一些随机错误， 可能是由于系统的 2 级高速缓存有问题。 严重的时候可能会导致数据完全丢失。这种情况下建议您关闭 2 级缓存。
        (3) 即插即用网卡和 16 位实模式驱动程序
        如果您需要为 NIC(NIC 比 DLC 或其它实模式协议更为常用) 运行 16 位实模式驱动程序，您的即插即用卡可能无法正常工作，因为在 Windows 95 打开即插即用卡之前， 16 位的 NIC 驱动程序就已经加载了。同样，一些 16 位的 NIC 驱动程序不能识别即插即用卡(大多数的 NE2000 即插即用卡均属于此类)。
        使用带 16 位 NIC 驱动程序的即插即用卡:
        ①运行即插即用卡自带的 Softset 实用程序，将之设置为非即插即用模式。
        ②双击“控制面板”中的“网络”图标，然后删除网卡。
        ③双击控制面板中的 “添加新硬件”  图标，  然后按屏幕上的指导进行。Windows 95会自动检测网卡。
        如果以后想改为 32 位保护模式的 NIC 驱动程序，可以再次运行 Softset 打开即插即用模式。
        (4) Intel EtherExpress 16 NICs 和 PCI 计算机
        如果在带有 Diamond Speed Star PCI 显示卡的 PCI 计算机上使用 IntelEtherExpress 16 位 NIC，您可能会遇到系统挂起或初始化方面的问题。以下内容选自 Intel 的客户支持数据库。 这些问题与 Windows 95 无关，在不同的操作系统平台上均可能出现。 总而言之， 如果在 PCI 计算机上遇到有关 EtherExpress16的其它问题，请在把问题报告给 Microsoft 之前试试替换该卡。
        如果有下列显示卡，应该更换新的视频 BIOS。
        - Diamond Speed Star PCI 显示卡， BIOS 1.01 版
        - Diamond Viper PCI VGA 显示卡
        - Diamond Stealth 显示卡，BIOS 1.03 版
        (5) IBM Thinkpad 型号 750X/755X/360X
        如果有下列型号的 Thinkpad(包括 C、 CE、 CX、 CS 和 CSE) ，应在安装Windows 95 之前升级更新的 BIOS 和 Mwave 驱动程序。 如果没有升级 BIOS，在计算机打开或关闭挂起模式时移动鼠标可能会出现问题。

        (8)防病毒软件
        某些防病毒软件可能会导致安装失败，所以在安装 Windows 95 前必须确保没有运行任何防病毒程序。如果程序是驻留内存的，请删除 Autoexec.bat、 Config.sys和 Win.ini 文件中所有对它的引用。
        如果 BIOS 带内置的病毒防护模块，应该在运行安装程序之前使用 CMOS 设置程序禁用它。若机器中有防病毒卡，在安装前最好从计算机中拨出。
        ①Central Point Anti-Virus 1.1 版 (Vsafe.com/Vsafe.sys)
        删除 Autoexec.bat 文件中所有含 Vsafe.com 或 Vsafe.sys 的行，同时也要将Win.ini 文件中所有调用 Vsafe.com 或 Vsafe.sys 的 "RUN=" 行删除。如果您是升级到 Windows 95， 那么， 在运行安装程序前， 必须把所有 VSAFE  创建的Chklist. ms文件删除。  由于  Windows  要为内存交换使用磁盘，  所以在运行Windows 95 时应该关闭Vsafe 的写保护选项。
        如果在安装 Windows 前没有删除 VSAFE，在启动 Windows 时您将看到有关Command. com、Win.com 和 Krnl.386 的警告信息。您可以忽略它们并选择更新选项。
        ②F-PROT Professional
        如果运行了 F-PROT Professional，安装程序将无法运行。请将其从Autoexec.bat 和 Config.sys 文件中删除，然后检查 Win.ini 文件中是否有下面这条语句:
        load=wvirstop.exe
        如果看到这一行，请删除它。完成以上操作后，请重新启动计算机，再次运行安装程序。
        ③Norton Anti-Virus(R) (所有版本)
        如果运行了任何版本的 Norton Anti-Virus，请在运行安装程序之前删除它。建议不要在 Windows 95 下运行 Norton Anti-Virus。
        删除 Norton Anti-Virus 1.5 版的方法:
        a. 在 Config.sys 文件中删除下面这一行:
        device=c:\nav\nav_.sys
        b. 在 Autoexec.bat 文件的 PATH 语句中删除 C:\NAV。
        删除用于 Windows 3.0 的防病毒特性的 Norton Desktop的方法:
        a. 在 Config.sys 文件中删除下面这一行:
        device=c:\ndw\navtsr.exe
        b. 在 Autoexec.bat 文件中删除下面这一行:
        c:\ndw\image.exe
        并且在 PATH 语句中删除 "c:\ndw"。
        c. 在 system.ini 文件中删除下面几行:
        shell=c:\ndw\ndw.exe (在 ［boot］ 部分)
        device=symevent(在 ［386enh］ 部分)
        d. 在 Win.ini 文件中删除下面这一行:
        load=c:\ndw\navtsrw.exe
        (9) 使用 TSR 程序和驱动程序
        ①APPEND (MS-DOS)
        不要使用 Windows 95 的 APPEND 实用程序。它将阻碍 Windows 和Windows下的程序为它们所用的文件创建合法的 WFP 名称。 (WFP 即文件的完整名称，包括驱动器名和从根目录开始的完整路径。)
        ②Autocon 2.0e 版、Bootcon 1.60 版和 Boot.sys
        启动计算机的时候，这些程序能让您选用不同的配置。 Windows 95 的安装程序只更改 Config.sys 和 Autoexec.bat 的开始部分。所以，如果您在安装完成后想使用这些程序，则必须手动更改这些文件。详细信息，请参阅该程序的文档。
        ③NetWare(R) NetBIOS
        NetWare (R) NetBIOS TSR 的大多数配置与 Windows 95 不兼容。多数情况下安装程序会检测到  NetWare  NetBIOS  并删除启动命令。  然而，  如果不是在Autoexec.bat中设置 NetWare NetBIOS(例如在另一个批处理文件中设置)，安装程序可能检测不到它。 此时您需要手工删除 NetWare NetBIOS。Windows 95 包含IPX/SPX 兼容协议，您可以用它代替 NetBIOS。
        ④用于 Widnows 的 Norton Desktop (Ep.exe)
        如果通过 Autoexec. bat 加载数据恢复程序 Erase Protect (Ep.exe)，最好在安装 Windows 前禁用它，  特别在您从早期的  Windows  版本中升级的时候。Erase Protect 将试图保存所有已删文件的副本，这可能会造成运行安装程序时磁盘空间不够。
        建议运行 Erase Protect 时使用永久交换文件。 如果使用临时交换文件，需要在删除保护中排除带 .tmp 或 .swp 扩展名的文件。
        ⑤Norton  Disklock 3.01 和 3.5 版
        在 Windows 95 下 Norton Disklock 3.01 和 3.5 版无法正常运行。 或者在运行Setup 之前删除该软件，或者与 Symantec 公司的产品支持部联系，索取与Windows 95 兼容的 Disklock 版本。
        ⑥PC Tools Deluxe(TM) 7.1 版(DATAMON)
        如果磁盘空间不够， 而且使用了 \SENTRY+ 选项加载 PC Tools DATAMON，安装程序将无法运行。 建议加载 DATAMON 时不要使用  \SENTRY+  选项。  在加载SmartDrive后不再加载 DATAMON，否则将会死机。请参阅 PC Tools 文档以获得如何卸载DATAMON的信息。
        ⑦PC Tools Deluxe 6.0 版 (DESKTOP)
        如果使用 Windows 的 MS-DOS 方式启动 PC Tools，可能会导致死机或自己重启动。 通过 Windows 的 MS-DOS 方式启动其它 MS-DOS 命令也会出现同样的问题。
        PC Tools 7.0 版可能解决了这些问题。然而，如果在启动 Windows 前已经加载了DESKTOP，请不要通过 MS-DOS 命令行加载。否则当您退出 Windows 时会死机。建议使用 Windows 95 提供的 PIF 运行 DESKTOP。  ⑧SUBST (MS-DOS)
        在安装过程中可能会遇到 SUBST 问题， 不过您可以在 Windows 95 安装完后安全地添加 SUBST。
        ⑨UNDELETE
        如果使用 SENTRY 选项运行 UNDELETE， 安装程序可能会出现问题，而且也可能在安装过程中出现磁盘空间不够的情况。 建议运行安装程序之前删除  Config.sys和Autoexec.bat 文件中的 UNDELETE。
        注意: SENTRY 选项会创建一个名叫 "Sentry" 的隐含目录。 删除文件时，该文件会被移至 Sentry 目录。 在删除 Config. sys 和 Autoexec. bat  文件中的UNDELETE之前， 请对所有驱动器使用 UNDELETE /PURGE 命令释放磁盘空间。如果您没有删除，安装程序会自动禁用该命令。
        (六) 快速安装WINDOWS 95中文版的方法
        从软盘上将 Windows 95 安装到内存为 4MB 或 386 计算机上时，速度可能会很慢。加快安装速度的办法:
        - 在 Windows 3.x 或 Windows for Workgroups 下运行安装程序。避免在MS-DOS 下运行安装程序。
        - 如果运行了磁盘压缩程序，如 DriveSpace 或 DoubleSpace，安装可能会很慢。
        可以考虑将软盘上的文件复制到硬盘的一个临时目录下，然后再从硬盘上运行安装程序。需要用 1 号安装盘上的 EXTRACT 命令将 2 至 12 号盘上的 WIN95_xx.CAB文件复制到硬盘上，因为它们是 Microsoft 的新的分布式媒体格式(DMF) 。一般的MS-DOS 命令，诸如 COPY 和 XCOPY，都无法读取这些软盘。
        将 .CAB 文件从软盘复制到硬盘，可以使用如下命令:
        extract /c a:win95_xx.cab c:\<TempDir>
        其中 EXTRACT. EXE在WINDOWS 95第一张系统盘上,<TempDir> 是硬盘上的临时目录。
        在安装过程中，应注意以下两个问题：
        - 安装过程中，如果注意到软驱中未插盘，而指示灯长时间闪烁且安装缓慢，请试将一张格式化好的软盘插入软驱。
        - 如果在安装过程中遇到问题，比如安装程序异常退出或出现未知错误，
        可以试试增加永久交换文件的大小。 通常永久交换文件的大小加上 RAM 的大小至少应该有 14MB， 这样才能顺利完成安装。请在更改交换文件大小后，再次运行安装程序。
        注意: 如果硬盘是压缩的，在增加交换文件大小之前可能需要先增加主驱动器的大小。

        六、 在WINDOWS  95中文版使用常见打印机
        (一)通过网络打印
        默认情况下， Windows 95  使用网络路径作为网络打印机的端口名。  这与Windows 3.1 略有不同，因此有些程序可能会有问题。典型的网络路径(又称 UNC名)具有以下格式:\\server\printqueue。
        然而， 网络路径往往比 Windows 3.1 中对应的名称要长，这在一些程序中会引起问题。 如果遇到了问题，可以采用 Windows 3.1 的网络资源寻址方法。
        如果网络路径中包含空格，有些程序也会遇到麻烦，您可以按前面所述的方法来解决问题，也可以更改网络路径使之不含空格。请向网络管理员求助以更改网络路径名。
        (二)在WINDOWS  95中使用佳能彩色 BUBBLE-JET 打印机
        对于佳能的彩色 BUBBLE-JET 打印机，如果它在 Windows 95 中的驱动程序是由 Windows 3.1 升级而来， 您可能会发现打印情况有所不同。由于 Windows 3.1的驱动程序在 Windows 95 中无法工作，佳能的BJC-70、BJC-600、BJC-600E、BJC-800 及 BJC-4000 打印机驱动程序在安装Windows 时将自动升级为  Windows  95的驱动程序。
        (三)在WINDOWS  95中使用对惠普 Deskjet 打印机
        惠普 DeskJet 打印机驱动程序 6.1 版和更早的版本在 Windows  95下无法工作， 也不可能安装。 如果您在 Windows 3.1 下使用的是这些驱动程序， 它们在Windows 95 下将被升级为 6.5 版。 如果您使用的是DeskJet 630 或 800 系列，请参考下面的的内容。
        1.惠普 DeskJet 600、660C 和 850C 打印机
        为了更有效地使用 DeskJet 600、660C 或 850C，应该先确定该打印机是否带有与 Windows 95 兼容的驱动程序 (要确定打印机驱动程序是否与Windows 95 兼容，请参考其磁盘上的标签)。
        情况 1: 如果有与 Windows 95 兼容的打印机驱动程序
        如果已安装了 Windows 95，同时要安装新的 DeskJet 600 或 800系列打印机，Windows 95 将自动识别打印机类型并安装驱动程序。在提示要求 OEM 提供的磁盘时，单击“确定”，然后插入打印机自带的HP DeskJet Printer disk 1 号盘，按照屏幕上的提示操作。
        如果 Windows 95 未检测出 HP  DeskJet  打印机，  请按照打印机自带的HPDeskJet Printer Driver disk 1 号盘标签上列出的步骤进行安装。
        如果从 Windows 3.1 升级  Windows  95，  而且已经为  DeskJet  安装了与Windows 95 兼容的打印机驱动程序(7.0 版或更高)，就不必再采取任何措施了。
        情况 2: 如果没有与 Windows 95 兼容的打印机驱动程序
        (只有 Windows 3.1 专用的驱动程序)
        要得到 DeskJet 打印机所用的与  Windows  95  兼容的驱动程序，  应与HPDeskJet 打印机用户手册中提到的惠普驱动程序发布中心联系，在得到Windows 95下的打印驱动程序之后，按下列步骤执行:
        ①单击“开始”按钮，指向“设置”，然后单击“打印机”。
        ②单击“增加打印机”图标，然后按屏幕指导操作。
        ③在出现提示要求选取打印机厂商和型号时，如果是 660C 或 850C
        打印机，则选“HP DeskJet 550C Printer”；如果是 DeskJet 600
        则选“ HP DeskJet 540”。
        ④单击“下一步”，按屏幕上指导进行操作。
        (四)  在WINDOWS  95中使用惠普 LASERJET 打印机
        如果从 Windows 3.1 升级到 Windows 95，您可以不考虑下面的内容；因为在Windows 95 安装过程中，所有经过优化的软件都已安装到计算机中。
        如果要增加一台 LaserJet 4 打印机、彩色 LaserJet 或 LaserJet 5P/5MP，按照下面所述的步骤可以获得最佳的打印机性能。
        1.惠普 LaserJet 4 打印机 - PCL 支持
        推荐软件 : Windows 95 打印机驱动程序和惠普的 TrueType 屏幕字体。
        惠普公司向 LaserJet 4 打印机的用户提供了惠普的 TrueType 屏幕字体，与您所用的内部打印机字体相匹配。
        2.惠普 LaserJet 4Si、4L、4P、4 Plus 和 4V 打印机 - PCL 支持
        推荐软件: Windows 95 打印机驱动程序和惠普的 True Type 屏幕字体。
        下面就是惠普激光打印机在WINDOWS  95中配置：
        ①将惠普 LaserJet Printing System 软盘中的 1 号盘插入到合适的软驱中。
        ②单击“开始”按钮，然后单击“运行”。
        ③在“运行”对话框中，键入 A:\Setup。
        ④单击“定制”安装，然后单击合适的惠普激光打印机。
        ⑤单击“HP TrueType 屏幕字体”。取消对打印机驱动程序、状态窗口以及执行指南的选择，单击“确定”，然后按屏幕的指导操作。
        3.惠普 LaserJet 5P 打印机 - PCL 支持
        推荐软件: Windows 95 打印机驱动程序和惠普的 TureType 屏幕字体注意: 需要使用惠普的 TrueType 屏幕字体与打印机内部字体相匹配，请找到惠普 LaserJet 附带的软盘，然后按下列步骤执行。
        ①将 HP 5P LaserJet Printing System 的 1 号盘插入到合适的软驱中。
        ②单击“开始”按钮，然后单击“运行”。
        ③键入 A:\Setup
        ④选“典型”安装，然后按屏幕上的指导操作。
        4.惠普的彩色 LaserJet 打印机 - PCL 支持
        推荐软件: 惠普的彩色 LaserJet 打印机驱动程序和惠普的 True Type 屏幕字体。
        注意: 请找到惠普彩色激光打印机附带的软盘，然后执行下列步骤。
        ①将惠普的彩色 LaserJet Printing System  (Windows 3.1 PCL  PrintingSystem)中的 1 号盘插入到合适的软驱。
        ② 单击“开始”按钮，然后单击“运行”。
        ③键入 A:\Setup
        ④ 选“快速”安装，然后按照屏幕上的步骤操作。
        注意: 要使用 Windows 95 中提供的图象颜色匹配功能， 需要安装  Windows95所带的惠普彩色激光打印机驱动程序。
        5.惠普 LaserJet 4M 到 5MP 打印机 - PostScript 支持
        推荐软件: Windows 95 PostScript 打印机驱动程序。
        注意: 不要使用惠普打印机所提供的 PostScript 打印机驱动程序。
        6.惠普彩色 LaserJet 打印机 - PostScript 支持
        推荐软件: 惠普彩色激光打印机 PostScript 驱动程序。
        注意: 找到惠普彩色 LaserJet 所带的软盘，然后按下列步骤执行。
        ①将惠普彩色 LaserJet Printer disk 中的Windows 3.1 PostScript 2.1.1盘插入到合适的软驱中。
        ②单击“开始”，指向“设置”，然后单击“打印机”。
        ③单击“增加打印机” 图标，然后按照惠普彩色 LaserJet PostScript驱动程序盘的使用说明执行。
        ④单击“从磁盘安装”按钮。
        如果询问是否要替换或更新驱动程序，请单击“是”。
        7.惠普 LaserJet 疑难解答
        (1)打印机状态冲突
        Microsoft 打印机状态报告程序和惠普状态窗口之间可能会有冲突。其现象是:打印过程中，Microsoft 状态程序提供打印状态信息，而其它时候，如果某台打印机故障或发生其它情况(如打印机缺纸、塞纸、并行电缆断开、打印机顶盖打开等等)，惠普状态窗口会向您报告相应信息。
        如果已安装了惠普状态窗口，可使用下列步骤将其卸载:
        ①单击“开始”，然后单击“程序”。
        ②单击“惠普 LaserJet”文件夹。
        ③双击“卸载惠普状态窗口”图标。
        ④单击“是”，重新启动 Windows。
        (2)“错误 21 _ 打印任务过重”
        如果 LaserJet 打印机发生这种情况，而且打印机属性的“设备选项”标签下“页面保护”特征可用，请打开该特征。必须在 LaserJet 3 打印机的前端面板上直接设置此选项。如果使用的是 LaserJet 4 打印机，且没有“页面保护”选项，可以在打印机属性的图形标签中将打印驱动程序由矢量方式改为光栅方式，或者降低该页的复杂度。
        (六)在WONDOWS  95中使用LEXMARK 打印机
        在 Windows 95 下， 当您使用某些 Lexmark 40X9 LaserPrinter 打印机的双向打印功能时可能会遇到问题。要启用双向支持，请按下列步骤操作:
        ①确认在 Windows\System 文件夹中有一个名为 LEX01.386 的文件。
        ②在 Windows 文件夹中的 System.ini 文件的 ［386 Enhanced］ 部分，
        加入以下行:
        *deviece=lex01.385
        WPSLPT#=1
        这里 # = 1、2 或 3 ，分别对应于 LPT1、LPT2 或 LPT3。
        (例如，对于 LPT2，此行应为 WPSLPT2=1)
        ③保存修改过的文件，重新启动计算机。
        七、在WINDOWS  95中运行MS-DOS和WINDOWS 3.1程序的一些方法及注意点
        1.在WINDOWS  95中文版中运行的MS-DOS应用程序
        MS-DOS实用程序在安装WINDOWS   95时将被安装在\Windows\Command目录中。 因为WINDOWS  95采用了长文件名，以前版本的磁盘修复程序不应在WINDOWS  95中使用。绝大多数在WINDOWS  3.1中工作的MS-DOS其它实用程序也可在WINDOWS  95中运行，但它们若是MS  DOS老版本的程序，则必须用SETVER.EXE程序来加载它们。
        下面的MS  DOS实用程序在升级到WINDOWS  95时将被删除掉：
        Defrag.hlp、Mwundel.exe、Mwundel.hlp、Networks.txt、OS2.txt
        下面的MS  DOS实用程序在WINDOWS  95升级时也自动升级：

        │  Debug.exe、Deltree.exe、Diskcopy.com、Display.sys、Doskey.com │

        │  Drvspace.exe、Emm386.exe、Ramdrive.sys、Readme.txt、Setver.exe│

        2.Microsoft FORTRAN 5.1在WINDWOS  95中运行
        若您要在WINDOWS  95中用Microsoft FORTRAN 5.1  建立QuickWin应用程序，则您必须在F1.DEF文件或是删除，或是说明下面一行：
        APPOADER '___MSLANGLOAD'
        在您删除了该行后，就必须用Microsoft FORTRAN 5.1重新编译QuickWin程序，以使这些程序能在WINDOWS  95中正确运行。
        3.PCTools Versions 7 -9在WINDOWS  95中运行
        在WINDOWS  95中安装PCTOOLS  version 7-9期间，热键将不能正确建立。为了在安装后能建立这些热键，可在WINDOWS  95中执行下面的步骤：
        (1)点按“开始”菜单，然后选择“运行...”。
        (2)键Cpsdos.grp文件名，再按回车键。
        此时， PCTools将被增加到程序的文件夹这。为了使热键能工作，您必须把PCTOOLS目录增加到PATH指定的搜索路径中。
        由于WINDOWS  95文件系统的改变，则PCTOOLS的这些磁盘实用程序可能对于长文件名不能很好地工作。您应该使用专为WINDOWS  95设计的磁盘管理程序。在WINDOWS  95中提供一些磁盘管理程序，例如SCANDISKW.EXE、DEFRAG.EXE等，它们能对长文件名进行良好地处理。
        同样地，对于NORTON  6.0-8.0的实用程序也应按上述方法处理。
        4.Personal Oracle 7.1 for Windows 3.1
        由于Personal Oracle 7.1 for Windows  3.1需要一个WIN32S.DLL的32位保护模式DLL程序，因此，它不能在WINDOWS  95中进行安装。您应该使用WINDOWS  95升级版Personal Oracle 7.1 for WINDOWS  95。
        八、WINDOWS  95中文版的CONFIG新配置介绍
        在WINDOWS  95中文版中CONFIG.SYS中配置有了一些新的变化，我们在此为读者解释一下这些新的变化。
        在WINDOWS    95中CONFIG.SYS文件可以出现的命令如下(打*号为新增或增强命令)：
        ACCDATE         (*)
        BREAK
        BUFFERS/BUFFERSHIGH  (*)
        COUNTRY
        DEVICE/DEVICEHIGH
        DOS
        DRIVPARM
        FCBS/FCBSHIGH        (*)
        FILES/FILESHIGH           (*)
        INSTALL/INSTALLHIGH       (*)
        LASTDRIVE/LASTDRIVEHIGH   (*)
        NUMLOCK
        REM
        SET
        SHELL
        STACKS/STACKSHIGH   (*)
        SWITCHES
        下面介绍一下WINDOWS  95中新增或增强的CONFIG配置命令。
        1.ACCDATE
        该命令为新增的CONFIG配置命令， 它配合了DIR/V命令。当您在WINDOWS  95的DOS窗口中执行DIR/V命令后，它将出现这样的信息：
        File Name  Size  Allocated  Modified  Accessed  Attrib
        Drvspace bin 70455  73728   07-07-95  09-09-95   A
        .....................
        其中ACCESSED列为记载了该文件最后存取的时间和日期，实际上，该项就是在CONFIG.SYS中ACCDATE命令配置形成的。
        该命令的语法为：
        ACCDATE=drive1+|- ［drive2+|-］...
        其中：drive1, drive 2...：指定驱动器的盘符，如A，B等。
        +|-： ＋号表示应该记录该驱动器上的文件存取的最后时间和日期，而-号与之相反。默认状态为＋号。
        例子：ACCDATE=A＋|B＋
        2.BUFFERS/BUFFERSHIGH
        BUFFERSHIGH是WINDOWS  95新增加的命令，它是对BUFFERS的增强。它可将WINDOWS  95系统的缓冲区设置到UMB中，这是对MS  DOS  6.22版本的一种改进。
        BUFFERSHIGH的语法为：
        BUFFERSHIGH=n［,m］
        其中的n,m含义同BUFFERS中的意义。n=1-99，缺省为30。m=0-8，缺省为0(不需要第二个缓冲区)。
        3.FCBS, FCBSHIGH
        FCBSHIGH是WINDOWS  95新增加的命令，它是对FCBS命令的增强。
        FCBSHIGH的语法为：
        FCBSHIGH=x
        其中x的含义同FCBS=x中的含义。x=1-255，默认值为4。
        4.FILES/FILESHIGH
        FILSHIGH是WINDOWS  95新增加的命令，它是对FILES命令的增强。
        FILEHIGH的语法为：
        FILESHIGH=x
        其中参数x的含义同FILES=X的意义。X=8-255，默认为8。
        在WINDOWS  95中文版中，FILESHIGH最好设置为50。
        5.INSTALL/INSTALLHIGH
        INSTALLHIGH是WINDOWS  95新增加的命令，它是对INSTALL命令的增强。
        INSTALLHIGH可把MS  DOS的驻留程序，例如Fastopen, Keyb,Nlsfunc, and Share放到UMBS中。
        INSTALLHIGH的语法为：
        INSTALLHIGH=［drive:］［path］filename ［command-parameters］
        其中的参数同INSTALL命令。
        6.LASTDRIVE/LASTDRIVEHIGH
        LASTDRIVEHIGH是WINDOWS  95新增加的命令，它是对LASTDRIVE命令的增强。
        LASTDRIVE的语法为：
        LASTDRIVEHIGH=x
        其中x=A-Z，意义同LASTDRIVE=x。
        7.STACKS/STACKSHIGH
        STACKSHIGH是WINDOWS  95新增加的命令，它是对STACKS命令的增强。
        STACKSHIGH的语法为：
        STACKSHIGH=n,s
        其中的参数n、s的意义同STACKS=n，s。n=8-64，m=32-512。 在WINDOWS  95中N、S=0是非法的数值。这一点与MSDOS以前版本是不同的，应引起读者的注意。
        九、Windows  95中文版使用技巧
        1.MS  DOS方式窗口中的技巧
        (1)MS  DOS方式下的可视目录显示
        在WINDOWS  95的MS  DOS方式下，若键入“SATRT.”或“START  ..”，则可在当前桌面上看到文件夹或当前目录。
        (2)MS  DOS方式下的拖放文件名
        您可从桌面上文件夹图标或文件夹中拖放(DRAG  AND  DROP)一个文件名到MS  DOS方式下的提示符下，作为运行MS  DOS程序的应用程序。
        (3)从MS  DOS方式拷贝或粘贴信息
        在MS  DOS应用程序窗口激活工具条，将WINDOWS应用程序中的有关信息拷贝、删除到剪贴板中，然后将它们粘贴到MS  DOS应用程序中。例如，可把文件夹在屏幕的输出拷贝到剪贴板中，然后，把它粘贴到MAIL中去。
        (4)从MS  DOS方式下启动一个WINDOWS应用程序
        您可从MS  DOS方式下直接键入WINDOWS应用程序的名字，然后指定它所需要的参数，就可运行了。例如，为了在MS  DOS方式下运行NOTEPAD的记事本WINDOWS程序，只要在MS  DOS提示符下键入NOTEPAD，然后按回车键即可。您也可以用批处理文件的方式启动WINDOWS应用程序。
        另外，您也可以用START命令来启动一个程序或打开一个文档，例如：
        C：〉START  Docname.ext。
        (5)Command.pif
        若您为MS  DOS方式设置了不正确的属性，而使您不能打开MS  DOS方式的窗口，则可进入PIF目录，然后编辑COMMAND.PIF文件，或找COMMAND.COM文件，在这里编辑其属性即可。
        (6)DOSKEY
        若在AUTOEXEC.BAT或启动脚本(SCRIPT)中未加载DOSKEY
        .COM程序，但又想在MS  DOS方式的窗口中使用它，则您可到“程序”属性中，输入DOSKEY作为启动批处理文件。若您后来决定您确实需要在启动脚本中加载DOSKEY程序，您只需将DOSKEY移到该批处理中即可。
        (7)目录的快捷键
        对于目录有以下的快捷键可供用户使用：
        .	= 当前目录	
        ..    	= 当前目录的父目录(上一级目录)	
        ...   	= 当前目录的上二级目录(WINDOWS  95新增加)	
        ....  	= 当前目录的上三级目录(WINDOWS  95新增加)

        2.删除、 拷贝、 粘贴、 拖、 放使用技巧(Cut, Copy, Paste, Drag andDrop)
        (1).拖和放(Drag and Drop)
        为了移动一个对象，可用鼠标器的左键从一个文件夹拖到同一个驱动器的另一个文件夹中。使用SHIFT＋鼠标左键则执行强制移动。
        为了拷贝一个对象，可用鼠标器的左键从一个文件夹拖到同一个驱动器的另一个文件夹中。使用SHIFT＋鼠标左键则执行强制拷贝。
        为了建立一个可执行文件的快捷键，使用鼠标器的左键拖一个可执行文件到一个新的文件夹中即可。
        用鼠标器的右键将一个对象从一个文件夹拖到另一个文件夹中， 然后， 选择“在这里建立快捷键”去建立此对象的一个快捷键。此时，该对象图标有以下几种含义：
        (1)“＋”加号为拷贝。
        (2) 箭头符号为一个快捷键。
        (3)无任何符号为移动。
        在拖动该对象过程中按ESC键将中止此操作。
        为了建立文档的快捷键，可用鼠标器的右键从一个OLE文档，例如WORD95中拖一个选择文档到桌面或文件夹上，然后选择“在这里建立文档快捷键”。于是，该文档快捷键将能立刻打开原始文档，即该快捷键已经赋予了此文档。
        (2).鼠标器的右键拖、放
        鼠标器的右键拖、放基本上同鼠标器的左键拖、放操作，只是它在放该对象的时候，将显示出一个菜单。该菜单用于文件夹，将显示出这些命令：“在这里移动”、“在这里拷贝”、“在这里建立快捷键”。对于OLE对象来说，该菜单还包括“在这里建立SCRAPS”、“在这里移动SCRAPS”和“在这里建立文档快捷键”等命令。
        (3).拖到任务条
        您能将对象从一个打开的文档拖到另一个打开文档上(即一个覆盖另一个)，此时，您只要做的事情就是把改该对象拖到目标对象的任务条即可。在目标对象的窗口显示的时候，您就可将此对象拖到目标窗口上。

        (9)重登录后的任务条的移去
        在您改变了WINDOWS  95的登录后的入口数据库后，若需要重新启动任务条，则可使用CTRL+ALT+DEL键将从WINDOWS  95中移去WINDOWS资源管理器。  当关闭系统的对话框出现的时候，按“NO”。在出现下一个提示时，按End Task。此时，任务条移去，然后，重新启动。
        (10)建立一个应急的启动盘
        为了建立一个完全应急启动盘， 可双击 “控制面板” 上的“增加/删除程序”图标，然后，点一下“启动盘”按钮即可。
        (11)取消一个文件移动或文件换名
        若您忘记了刚才所作的移动文件位置或意外地将某一个文件换名，可在“编辑”菜单中选取“取消”或按ALT+Backspace键就可将上次所作的操作作废。
        (12) 改变显示属性
        使用鼠标器的右键点一下桌面，再点“属性”，就能改变桌面上的背景、屏幕保护器、表面或显示器的设置等。为了改变您初始的显示字体，可选择“表面”按钮改变图标的字体。
        (13)使用任务条的时钟的技巧
        将鼠标器的指针放到任务条的时钟上，此时就在此下面可看到当前系统的日期。若双击此时钟，则将屏幕上出现一个较大的时钟画面，里面有系统时间、日期和时区等信息，它有点类似于WINDOWS  3.1的“附件”中时钟程序。
        为了从任务条中移去时钟，可执行以下步骤：
        ① 使用鼠标器的右键点一下任务条，然后，选取“属性”。
        ②按一下任务条上的“选项” 按钮，不选择“显示时钟”核实框，或按C键也可。
        (14) 易于获取计算机的属性
        为了查看计算机(系统)的属性，可使用鼠标器的右键点一下“我的电脑”，然后， 选取“属性” 。若您有一个Microsoft Natural Keyboard, 则只要按Win+Break就可以了。
        为了查看Network属性， 可使用鼠标器的右键点一下Network Neighborhood图标，然后，选取“属性”即可。
        (15)使用光标键代替一个鼠标器
        通过打开“鼠标键”可不使用鼠标器移动光标。为此，可双击“控制面板”中的“无障碍选项”图标，按“鼠标”按钮，然后，点一下“使用鼠标键”即可。
        (16)快速到达控制面板的简便方法
        ①使用鼠标器右键点一下“开始”菜单，再选取“资源管理器”。
        ②在“开始” 菜单中建立一个名字为Control Panels的新目录，或者您命名为其它名字也可。
        ③ 打开原始的“控制面板” 的文件夹，选择所有图标，再用鼠标器的右键点把它们拖到您的刚才建立的新的文件夹中。此时，就建立为“控制面板”中的所有图标建立了快捷键。
        ④关闭以上所有的窗口， 再点一下“开始”菜单。当您用鼠标器指向到您的新文件夹中，则您就看到了所有“控制面板”中的图标。
        (17) 通过使用“Send To”发送对象
        您可以把任何快捷键拖动到“SendTO”的文件夹中。这个快捷键可以是一个打印机、传真、网络驱动器或WINDOWS应用程序。为了找到SENDTO的图标，可点一下“开始”菜单，再选取“运行...”。在运行对话框中输入SENDTO命令，就启动了一个SENDTO文件夹。此时，您就可以使用鼠标器的右键点拖动某项目的图标到SENDTO窗口中，该图标的快捷键就将出现在SENDTO菜单中。使用鼠标器的右键点一下一个文档或文件夹图标，选取“SENDTO”，再选择目标项目，则就发送对象了。
        12.WINDOWS  95中的系统工具
        (1) 系统监视器
        系统监视器可监视系统资源、线程、处理器使用等。
        (2) 压缩程序
        建立或形成压缩磁盘并管理压缩磁盘等工作。
        (3)备份与恢复程序
        您可使用Backup备份或用restore恢复文件到磁带、软盘等。
        (4) 磁盘重整程序
        该程序通过重整工作将加快硬盘的速度。
        (5) 网络察看器
        该程序察看在您的计算机上哪些用户在使用您的资源和增加共享文件夹等。
        (6)磁盘扫描程序
        该程序将检查计算机的逻辑或物理错误，并将尽量修复它们。
        注意：以上程序可支持WINDOWS  95的长文件名。在WINDOWS95中最好不要使用以前MS  DOS版本或WINDOWS  3.1版本的磁盘诊断或扫描程序
