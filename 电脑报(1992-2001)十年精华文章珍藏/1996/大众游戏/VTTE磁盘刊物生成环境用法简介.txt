VTTE磁盘刊物生成环境用法简介

姚忠林
1996-06-28

        一、VTTE主要特点如下：
        无需任何汉字系统的支持，汉字文本文件(长度最长可达450K) 转换后可直接在西文DOS环境下执行。
        支持各种32点阵艺术字为有20余种字体，包括录书，行楷，魏碑等。
        支持67种不同英文字体。
        汉字文本中可嵌入单色图像文件(应用户要求可制作16色效果)。
        生成的可执行文件自动压缩，体积小，减少占用硬盘空间。
        支持各种修饰，可在文件中加入各种控制符对显示效果进行调整， 如设置前景色，背景色、屏幕颜色、下划线，反相显示，行间距等。
        提供全交互式版面设计环境，用户可边修改边观察效果，真正做到所见即所得， 同时内置全屏幕编辑器，方便用户手动修改。
        全面支持打印机，包括LQ系统和HP喷墨及激光打印机。
        提供图文并茂的目录检索与跳转功能，用户可自定义位置不限。
        数目不限，级数不限的目录结构，并可对某个词或句子加上单独的注释窗口。
        全面支持声霸卡系列，从8位到16位，可以在阅读时辅以背景音乐。
        自藏80余首乐曲的曲库。
        全开放性的体系结构，所有显示信息均可由用户定义，包括上下标题栏， 帮助信息等。
        全面利用Windows的图形资源，可直接利用ICO的图标文件及BMP 图象文件， 内含有1000余种各类图标的图标库。
        先进的底层级操作，直接对VGA寄存器编程，保证了极高的显示速度。
        提供了外调其他演示工具的功能。比如外调播放一个FL1或FLC二维或三维动画文件。
        二、VTTE 的运行环境配置
        IBM PC AT 兼容机、MS-DOS V3.30以上、不少于640K内存、 VGA显示器、4MB 以上的硬盘空间。(如需欣赏音乐，还需声霸卡)
        三、VTTE基本使用方法：
        VTTE提供了一个友好的图形用户界面，以便让用户方便快捷地掌握其基本用法。
        键入命令：CD\VTTE  进入包含Visual TTE的目录。
        键入命令：VTTE  则启动Visual TTE 主程序。这样便可进行文本文件的转换操作了。
        注意：如果我们是在中文环境下(如：UCDOS天汇等)键入VTTE命令， 则屏幕显示：本程序检测到当前环境是在汉字系统之下，本程序的显示可能会不正常。按任一键退出……。
        启动VTTE程序后，屏幕上显示图形界面
        VTTE可以使用下拉菜单进行选择各种要使用的菜单项，为了提高效率，可以用另一种操作方法，即快捷键操作方式，图形界面底行有快捷键的提示，更方便用户的使用。转换文件的操作步骤如下：
        首先按F3键，屏幕上弹出一个文件选择窗口供选择要转换的源文件，从中可以轻松自如地切换目录和驱动器，文件选择窗口由行输入域、文件列表框、目录列表框和驱动器列表框组成。可以在行输入域直接键入文件名，也可输入一组通配符(缺省值为*.txt)，然后从文件列表框中选取文件。按TAB键或→键可在各域之间进行切换，最后按OK 按钮进行确认，或按CANCEL按键取消。
        按下来按F4键，屏幕上弹出一个对话框供指定生成的可执行文件名( 缺省的文件名与源文件名相同，扩展名为*.EXE)，可以重新输入新的文件名，然后按OK按钮确认。
        然后按F5键，转换即刻开始，如果输入的可执行文件名已经存在，系统会弹出一个提示窗口询问是否要覆盖它，如果选择NO按钮，系统将提示重新输入执行文件名，否则将直接覆盖已存在的文件。
        转换过程中，屏幕会有一个窗口提示系统正在进行转换。如果出现任何的错误，系统将会弹出一个红色警告窗口，提示出错信息以便用户采取相应的措施，转换完毕后，系统会弹出一个绿色窗口提示转换已经完成。
        最后，可以直接按F8键观察生成文件的效果。也可以在DOS 下直接运行该可执行文件。
        VTTE系统错误信息列表：
        “General File I/O Failure(Undefined)”
        未定义的文件I/O错误，一般是磁盘已满或出现结构错误(可用NDD检查)
        “Required .SPT file not found”
        在源文件中指定的.SPT图象文件未找到
        “Required Lib File ASC32B not found”
        库文件ASC32B未找到
        “Required Lib File ASC32Q not found”
        库文件ASC32Q未找到
        “32X32 DOT Lib File xxx32 not found”
        在源文件中指定的某种32点阵字体的字库未找到
        “16x16 DOT Lib File H2kxsj not found”
        16点阵显示字库未找到
        “Required .CMF File not found”
        指定连入的CMF音乐文件未找到。
        “Required Special ASC Lib not found”
        必需的文件FNT.LIB未找到
        “Unable to creat Target .EXE file”
        无法创建指定的目标执行文件，一般是磁盘已满或出现了结构错误(可用 NDD检查)
        四、VTTE 控制符的使用
        为了能编辑出更优美，更直观的执行文件，Visual TTE增加了许多强大的功能。要充分利用系统本身强大的资源，必需了解并熟练掌握各种控制的使用。
          VTTE所使用的控制符都是由字符“`”(即键盘上数字键1左边的那个键)开始。下面就其功能作简单的介绍。
        1. 调节字符的显示颜色
        VTTE系统允许所显示的字符文字使用16种前景色和16种背景色，只需在字符前加入字符串`F［Color］和`C［color］即可。其中，`F［color］用来改变字符的前景色，`C［color］用来改变字符的背景色，控制符F、C必须大写(以下其它控制符也需大写)，而［color］是a到p的小写字母，其对应颜色如下：
        a 蓝色   b 黑色  c 绿色  d 青色   e 红色  f 紫色  g 棕色   h 浅灰
        i 暗灰   j 亮兰  k 亮绿  l 亮青   m 亮红  n 亮紫  o 亮黄   p 白色
        例如：要显示绿底红字的“电脑报”这三个字应该这样使用控制符`Fc`Cc 电脑报
        2.  改变底背景颜色(系统建议不随便使用)Visual TTE引进了背景底色的概念，这可以使用户更好的设计版面，使颜色搭配更加和谐。
        使用方法：`G［color］颜色定义同上，所要注意的是当指定某个颜色为底背景色时，则该颜色的编辑为a，而原来的编号对应的颜色则为黑色。 系统底背景颜色缺省值为蓝色。
        3. 设置行间距
        命令格式为`H【点数】。点数是用a到z小写字母表示。a表示0点，b表示1点，……z表示25点。
        4. 英文体的选择
        由于系统提供了67种英文字体供用户选择，因此系统提供三个控制符。
        命令格式如下：
        `X【字体号】 字体号为a-z，对应英文字体0__25
        `Y【字体号】 字体号为a-z，对应英文字体26__51
        `Z【字体号】 字体号为a-o，对应英文字体52__67
        注意：本控制符仅对一行英文字体有效。
        5. 关于32点阵汉字的显示。
        VTT E极富吸引力的一个特点就是它支持32点阵汉字的显示， 而且有多种字体可供选择，需要注意的是，一个32点阵的高度相当两个16点阵汉字，因此，其显示分为两行进行。这样做的好处还在于，在一个32点阵的汉字两旁可以同时显示上下两行16点阵汉字，便于制作上标和角标。
        命令格式如下：
        `U【TYPE】指定当前行显示32点阵汉字的上半部
        `D【TYPE】指定当前行显示32点阵汉字的下半部
        其中TYPE是32点阵汉字的这体类型。共对应字符如表格①
        命令`E：指定当前行为16点阵显示。
        例如要显示以下效果
        超级视频播器(SUPER PLAYER)
        (三通公司)
        控制命令：
        `Ua超级视频播放器`E(super player)
        `Da超级视频播放器`E(三通公司)
        6. 下划线开关
        命令格式：`L
        第一次使用为开，再一次为关。
        7. 字符反相显示
        命令格式：`I
        使用效果使当前字符显示反相。
        8. 调用.SPT图像
        在文件说明，产品介绍中往往需要插入各种图片。VTTE系统有调用.SPT图像的功能。
        命令格式：`P“文件名”【`Fcolor】【`Ccolor】
        此命令必须加文本文件的每一行的第一例，且P与文件名之间不可有空格，.SPT 文件必须是单色图象。【`Fcolor】【`Ccolor】这两个可选参数是用来定义图象显示的前景色和背景色，颜色代号同上，.SPT文件可由文字处理器WPS生成， 生成时注意图形不要超出640点宽，对其编辑可用WPS附带的图文处理器SPT，注意存盘时选用非压缩的SPT格式存盘，WPSNT由于对SPT格式做了较大的修改，因此系统建议使用6.0F版的WPS， 对于其它图形，可选用工具将其转换为TIF格式，再由SPT转换成SPT格式。另外，为了更好地利用上部资源，系统本身提供一种图形转换工具BMP2SPT，可将标准位图格式(.BMP)转换到本系统可使用的图象SPT格式。
        使用方法：BMP2SPT bmp文件名 Spt文件名 参数
        参数意义如下：
        /A 背景为白色，不进行反相处理
        /B 背景为白色，进行反相处理
        /C 背景为黑色，不进行反相处理
        /D 背景为黑色，进行反相处理
        若BMP为彩色图象，可利用Windows中的画笔以单色格式另存盘后，再用BMP2SPT 转换。
        9. 命令′：由于VTTE使用字符“′”作为控制字符，为了显示字符`， 必须使用“这种方式。
        10. 关于制作全局跳转菜单
        命令格式如下：
        `<菜单名1`> 标志目录索引项所对应的跳转位置
        例如：在文章目录区定义`<参考文献`>则屏幕上对应的位置会显示白底红字的：参考文献，在文章中写有参考文献的那一段上定义：`【参考文献`】 ， 这样在阅读到有白底红字的“参考文献”时，打TAB键，即可见当前屏幕上第一个用`< `>定义的标题反相显示，使用TAB键或方向键移动反相光标选定所要跳转的标题。 如选“参考文献”按回车键，好会跳转到文章中写有参考资料的那一段上去。
        值得用户注意的是，`< `>和`【 `】是一组对应的命令。不能一对多，例如`<备注`>对应两个`【备注`】，系统只承认第一个`【备注`】，但却允许多对一，例如在文章中多处出现`<备注`>打TAB键，按回车后系统都会跳转到`【备注`】的地方。另外控制符内的字符必须完全对应才能实现对应，例如`<备   注`>中间空三格，那么` 【备   注`】也必须空三格，否则不会跳转。
        11. 关于制作标题注释
        命令格式：`<标题`@注释内容(用@分行)`>
        12. 关于外调演示程序
        此功能用于在阅读时外调外部的演义程序
        命令格式：`<标题`@`S【文件名】`>
        在使用TAB键选中标题时DOSSHELL一个外部演示程序， 这个外部程序必须是一个非常驻内存程序。例如，要调用UCDOS中的DEMO.EXE演示程序，可以定义如下：`<UCDOS演示`@`SC:\UCDOS\demo.exe>
        13. 关于图标的制作
        Visual TTE支持图标，图标的点阵大小为32X32点阵与32点阵汉字等大， 所以在控制符仍采用双行表示法。命令格式如下：
        `M汉字
        `N汉字
        本系统所配图标有一千余种，为了在控制符中区分各种图标，系统采用了图标对应对字的办法，即每个图标都对应一个汉字，如使用
        `M啊
        `N啊
        即可显示一个图标
        关于图标与对应表可在DOS下执行SHOW.EXE文件，该文件执行后显示8×16个方程，每格格式如下：
        \[2612961]
        14. 制作上标题
        命令格式：@(超级可视磁盘…… F1帮助)
        在文件行的顶头加入，可以设置上标题框显示内容。
        15. 制作下标题
        命令格式：@【本说明文件…… 1996年8月】
        在文件行的顶头加入，可以设置下题示行显示内容。(注意长度应为80， 即一行，不足用空格补满)
        16. 制作帮助信息框
        命令格式：@<第一行@第二行@第三行>
        在文件行顶头加入，可设置帮助框显示内容。
        五、使用VTTE的配套工具
        为了更方便地使用Visual TTE，系统专门为用户设计了几个配套工具，熟练使用这些工具可以起到事半功倍的效果。
        TTEDIT是一个文本文件编辑器，使用这个工具可以减少控制符的记忆量，大大缩短颜色调试时间，减少工作量，并且在修改的同时直接观看显示效果，真正做到所见即所得。
        在系统中按下F7键，或在DOS下键入TTEDIT，然后输入欲转换的文本文件名，屏幕将分为两大部分，上半部分显示文本文件，下半部分是命令菜单
        在TTEDIT编辑器中可以用下半部分的各种编辑命令在文本文件中定位，用相应的菜单控制键进行效果修饰，并同时观察修改后的效果，对于设置各种颜色，32点阵字符十分方便，而且所见即所得，便于修改。如果这样编辑的效果还不能满意或想达到某种特殊的效果，可以按“E”键入进入全屏幕编辑器中自行调整，自动转换完毕后，要按“S”键存盘后，再退出。
        注：Visual TTEDIT所能处理的文本文件的最大长为32K，若文件长度超过32K， 请分割后分开处理以免出错。
        在VTTE系统中直接按F7键，即可进入TTEDIT编辑器。但由于VTTE不能在中文环境下运行，而TTEDIT却能在中文环境下运行，且在中文环境下更能发挥它强大的编辑作用。所以一般不采取在VTTE系统中按F7键进入TTEDIT的办法，而采取直接在DOS 的中文环境下单独运行TTEDIT编辑工具，若在VTTE系统中，可选择DOSSHELL即按ALT-D，切换到DOS命令状态，再进入中文环境，运行TTEDIT。首先进入的是可视的设置各种控制符命令的编辑器，可称其为“命令编辑器”，在此编辑器中进行各种控制符的使用和定位。按“E”键即可进入全屏幕编辑器，此全屏幕编辑器是一个纯文本文件的编辑器， 亦有块的功能等，相当于WPS中的非文书文件的编辑功能。 在中文环境下全屏幕编辑器可以进行汉字的输入和修改，然后按“ESC ”键即可退到命令编辑器中进行控制符命令的工作，若不满意按“E ”键又可进全屏幕编辑器中进行汉字修改和录入或进行特殊控制符命令的定位。如此反复，直到满意为止，再进入VTTE系统进行转换。
        总之，Visual TTE 是一个制作可视文本文件的优秀工具软件， 非常适合用户制作磁盘刊物、软件说明书等等，以上介绍的控制使用的基本知识，掌握程度主要靠用户平时多使用多练习，定能制作出图文并茂， 排版新颖生动的可视文本来。
