中高档微机CMOS参数的优化设置

何宗琦
1996-09-06

        第一，界面友好、操作方便。如WinBIOS设置程序提供了图形窗口界面，允许使用鼠标、键盘和光笔操作。
        第二，增加了各种硬件新技术的参数设置功能。如即插即用(PnP)、增强型IDE、局部总线、扩展数据输出(EDO)、电源管理等参数的设置，使系统配置更加灵活、方便。
        第三，提供了自动设置方式。一般用户对系统硬件和芯片参数了解不够，往往不能正确设置，影响了系统性能的发挥，甚至不能正常工作。采用自动设置可使系统达到较佳状态。
        本文介绍了这些新特点和参数的设置。应该说明的是，虽然不同主板的BIOS设置程序不同，但其设置的参数却有很多共同之处，只要了解这些参数的意义就可以作出正确的设置。
        一、增强型IDE(EIDE)参数设置
        增强型IDE(Enhaced IDE)是Western Digital为取代IDE而开发的新接口标准。在很多新型微机系统中，EIDE接口已直接做在主板上，可与EIDE设备直接连接，因此不必再购买单独的适配卡。与IDE相比，EIDE有以下几个方面的特点：
        ·支持大容量硬盘，最大容量可达8.4GB。而原有的IDE标准，因受到硬盘磁头数(最大为16)的限制，其管理的最大硬盘容量，不超过528MB。
        ·可连接更多的外设，最多可连接四台EIDE设备。原有的IDE只提供一个IDE插座，因而最多只能挂接两个硬盘。EIDE提供了两个接口插座，分别称为第一IDE(Primary)接口插座和第二IDE(Secondary)接口插座。每个插座又可连接两个设备，分别称为主(Master)和从(Slave)设备。因此一共可连接四台设备。第一接口插座也称为主通道，它通常与高速的局部总线相连，用于挂接硬盘等高速的主IDE设备(Primary IDE Drives)。第二接口插座称为辅通道，一般与ISA总线相连，可挂接CD-ROM或磁带机等辅IDE设备(Secondary IDE Drives)。在BIOS设置中，要求用户对Secondary IDE Drives的数量，主从设备的工作模式进行设置。
        ·EIDE标准允许支持除硬盘以外的其它外设。旧的IDE标准只支持硬盘，因此它只是一个硬盘标准。而EIDE支持符合ATAPI接口(AT Attachment Pactet Interface)标准的磁带驱动器和CD-ROM驱动器。因此我们在谈到IDE连接的对象时，只能说硬盘，而谈到EIDE连接的对象时就可笼统地说EIDE设备。
        ·EIDE具有更高的数据传输速率。原有的IDE驱动器的最大突发数据传输率(Burst Data Transfer Rate)仅为3MBps。突发数据传输率是指从硬盘缓冲区读取数据的速度，其单位常用每秒兆字节(MBps)或每秒兆位(Mbps)。EIDE支持SFFC(Small Form Factor Commitee)制定的新的宿主传输标准，如PIO(Programmed Input/Output)Mode 3以及PIO Mode 4，其突发数据传输率可达11.1MBps和16.6MBps；Multiword Mode 1 DMA以及Multiword Mode 2 DMA，其突发数据传输率为13.3MBps和16.6MBps。此外，还应说明两点。第一，只有硬盘和系统BIOS都支持EIDE及PIO或Multiword DMA标准，才能发挥其高数据传输率的性能。第二，为了说明不同的传输标准，通常把支持PIO Mode 3或Multiword Mode 1 DMA的系统和硬盘称为Fast ATA，而把支持PIO Mode 4 或Multiword Mode 2 DMA的系统和硬盘称为Fast ATA-2。理解上述概念对正确进行BIOS设置是必要的。
        下面介绍与EIDE有关的设置参数。在这些参数中，仍用了IDE这个名称，但实际上指的都是增强型IDE，即EIDE。
        1.HDD Mode设置硬盘工作模式
        EIDE支持三种硬盘工作模式：NORMAL、LBA和LARGE模式。
        ·NORMAL Mode普通模式
        这是原有IDE方式。在此方式下，对硬盘访问时，BIOS和IDE控制器对参数不作任何转换。在普通模式下支持的最大柱面数为1024，最大磁头数为16，最大扇区数为63，每扇区字节数为512。因此支持最大硬盘容量为：
        512×63×16×1024=528MB
        即使硬盘的实际物理容量更大，但可访问的硬盘空间也只能是528MB。
        ·LBA(Logical Block Addressing) Mode逻辑块地址模式
        这种模式所管理的硬盘空间突破了528MB的瓶颈，可达8.4GB。在LBA模式下，设置的柱面、磁头、扇区等参数并不是实际硬盘的物理参数。在访问硬盘时，由IDE控制器把由柱面、磁头、扇区等参数确定的逻辑地址转换为实际硬盘的物理地址。在LBA模式下，可设置的最大磁头数为255，其余参数与普通模式相同。由此可计算出可访问的硬盘容量为：
        512×63×255×1024=8.4GB
        ·LARGE Mode
        当硬盘的柱面超过1024而又不为LBA支持时可采用此种模式。LARGE模式采取的方法是把柱面数除以2，把磁头数乘以2，即使柱面数减少，但总容量不变。例如，在NORMAL模式下柱面数为1220，磁头数为16，进入LARGE模式则柱面数为610，磁头数为32。这样在DOS看来柱面数小于1024，即可正常工作。相反的转换进程由BIOS的INT 13H完成，以便取得正确的硬盘地址。
        LARGE模式支持的最大硬盘容量为：
        512×63×32×512=1GB
        用户可根据配置的实际硬盘在上述三种工作模式中选择设置。
        2.IDE BLOCK Mode设置IDE块模式
        IDE设备块模式是指在每次中断时都以特定的扇区数作数据传送。当配置的硬盘支持块模式时，可允许按此模式工作，以提高访问硬盘速度，否则应禁止按此模式工作，以避免硬盘访问出错。本参数的设定值在不同的BIOS版本中不完全相同，一般为AUTO/Optimal/Disabled。选择AUTO时，将按照硬盘自动检测功能的报告值作为数据传送的扇区数；若选Optimal则以最佳缺省设置值为该扇区数；若选Disabled则禁止本模式工作。
        3.32-Bit Transfer设置32位磁盘数据传送
        32位磁盘访问是Windows 3.1以上版本提供的一个功能，它允许Windows应用程序在保护模式下执行磁盘数据传送。
        本参数的设定值为Disabled/Enabled(禁止/允许)。
        应该说明的是，仅允许32位磁盘传送对系统性能的改善并无多少影响，还必须配合32位文件访问，才能提高系统性能。32位文件访问(32-bit file access)也是Windows 3.1以上版本提供的功能，它以32位保护模式高速缓存代替了SmartDrive的16位高速缓存，并允许Windows应用程序在保护模式而不是通常的DOS实模式下操作文件分配表(FAT)。为了激活32位文件访问功能，可打开控制面板，双击386增强方式图标，选择虚拟内存按钮，再单击更改，再单击32-bit文件访问开关项，当方框中出现“×”后，选择OK按钮即可。当两种32位访问激活后，性能可提高68%以上。
        4.PIO Mode设置PIO模式
        本参数用来设置IDE设备的PIO模式。前面已经介绍，PIO模式直接影响了数据传输速率，应根据设备的特性加以设置。本参数的设定值为AUTO/1/2/3/4。当选择AUTO值时，将根据自动检测硬盘所得的值来设定PIO模式，用户也可人工修改设定值。
        5.Secondary IDE Drives present设置第二IDE设备的数量
        这个参数用来设定与第二IDE接口相连的IDE设备的数量，以便通知系统检测和使用。该参数的设定值为None/1/2，用户应根据实际配置的数量设定。
        6.Onboard IDE或Onchip IDE设置主板上的IDE接口
        本参数用来设置是否允许使用主板上的IDE接口。设定值为Enabled/Disabled。有些版本的BIOS可用AUTO/作为本参数的缺省设定值，只要有IDE设备连接到该接口时，即允许该接口自动工作。
